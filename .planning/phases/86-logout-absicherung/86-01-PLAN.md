<objective>
Logout-Absicherung End-to-End: Das aktive Refresh Token wird beim Logout serverseitig revokiert, sodass ein abgegriffener Token wertlos wird.

Purpose: Ohne serverseitiges Revoke bleibt ein gestohlenes Refresh Token bis zum Ablauf (90 Tage) gueltig. Nach dieser Phase ist der Logout sicher.
Output: Backend-Endpoint POST /api/auth/logout + Frontend-Aufruf desselben vor clearAuth()
</objective>

```javascript
// POST /api/auth/logout — Revokiert das aktive Refresh Token
router.post('/logout', verifyToken, async (req, res) => {
  const { refresh_token } = req.body;

  // Best-effort: Logout geht immer durch, auch ohne Token
  if (!refresh_token) {
    return res.json({ message: 'Logout erfolgreich' });
  }

  try {
    const hash = hashToken(refresh_token);
    await db.query(
      'UPDATE refresh_tokens SET revoked_at = NOW() WHERE token_hash = $1 AND revoked_at IS NULL',
      [hash]
    );
    res.json({ message: 'Logout erfolgreich' });
  } catch (err) {
    console.error('Fehler beim Revoke des Refresh Tokens:', err);
    // Best-effort: Fehler nicht an Client weitergeben, Logout trotzdem bestaetigen
    res.json({ message: 'Logout erfolgreich' });
  }
});
```

Wichtig:
- verifyToken Middleware sichert den Endpoint (kein ungeschuetzter Massenloesch-Endpoint)
- Best-effort-Muster: Logout scheitert nie, auch wenn DB-Query fehlschlaegt
- Kein 401 zurueckgeben, auch wenn kein refresh_token mitgeschickt wird — App muss sich immer ausloggen koennen
