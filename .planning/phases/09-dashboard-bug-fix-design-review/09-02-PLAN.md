**Teil B: DashboardView.tsx ueberarbeiten**

1. **Tageszeitabhaengige Begruessing** (LOCKED DECISION):
   Hilfsfunktion VOR dem DashboardView-Component hinzufuegen:
   ```typescript
   const getGreeting = (name: string): string => {
     const hour = new Date().getHours();
     let greeting: string;
     if (hour < 12) {
       greeting = 'Guten Morgen';
     } else if (hour < 18) {
       greeting = 'Guten Tag';
     } else {
       greeting = 'Guten Abend';
     }
     return `${greeting}, ${name}!`;
   };
   ```
   Die h2-Begruessing (aktuell `Hey {getFirstName(...)!`) ersetzen durch:
   ```tsx
   <h2 className="app-dashboard-greeting">
     {getGreeting(getFirstName(dashboardData.konfi.display_name))}
   </h2>
   ```

2. **Header Card:** Die aeussere div (Zeile 569-577) ersetzen durch `className="app-dashboard-header"` und den style-Block entfernen. Die dekorativen Kreise (Zeilen 579-614) auf `className="app-dashboard-header__circle"` umstellen, aber die individuellen Positionswerte (top/right/left/bottom/width/height) als style beibehalten (da sie einmalig und positionsspezifisch sind). Der erste Kreis bekommt zusaetzlich `background: rgba(255, 255, 255, 0.08)` (etwas heller) als style.

3. **Subtitle Jahrgang:** `className="app-dashboard-subtitle"` statt inline styles.

4. **Level Badge:** `className="app-dashboard-glass-chip"` statt inline styles (Zeile 658-668).

5. **Alle 5 Sektionen** (Konfirmation, Events, Tageslosung, Badges, Ranking):
   - Aeussere div: `className="app-dashboard-section app-dashboard-section--{name}"` statt inline style
   - Background-Text div: `className="app-dashboard-section__bg-text"` statt inline style
   - Background-Label h2: `className="app-dashboard-section__bg-label"` statt inline style
   - Content div: `className="app-dashboard-section__content"` (oder `--compact` fuer Events/Ranking) statt inline style

6. **Tageslosung Zitat-Style** (LOCKED DECISION -- Balken links):
   Das blockquote (Zeile 1091-1101) ersetzen:
   ```tsx
   <blockquote className="app-dashboard-quote">
     "{text}"
   </blockquote>
   <cite className="app-dashboard-cite">
     {reference}
   </cite>
   ```
   Statt zentriertem Text wird der Vers nun linksbuendig mit einem vertikalen Balken links dargestellt -- das unterscheidet die Tageslosung visuell von anderen Sektionen.

7. **Dunkle Punkte als Detail-Element** (LOCKED DECISION):
   In Sektionen wo Meta-Informationen nebeneinander stehen (z.B. Datum + Ort in Konfirmation und Events), zwischen den Items den Punkt-Separator verwenden:
   ```tsx
   <span className="app-dashboard-dot" />
   ```
   Konkret: In der Konfirmation-Sektion zwischen Datum und Ort. In der Events-Sektion zwischen Datum und Uhrzeit. In der Ranking-Sektion zwischen Platzierung und Punkte-Label.

8. **Event-Items:** Die Glass-Cards in der Events-Sektion (Zeilen 932-947) bekommen `className="app-dashboard-glass-card"` als Basis, behalten aber dynamische border-Styles (Warteliste/Abgesagt) als inline style.

9. **Legende CSS-Variablen** (LOCKED DECISION): In ActivityRings.tsx (NICHT in diesem Plan, dort schon bearbeitet) sind die Farben korrekt. Die Legende in DashboardView selbst nutzt die Farben aus ActivityRings -- nichts zu aendern hier, da ActivityRings die Farben definiert.
