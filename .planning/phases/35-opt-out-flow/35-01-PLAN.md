  <files>init-scripts/01-create-schema.sql, backend/routes/konfi.js</files>

1. **Schema-Datei** (`init-scripts/01-create-schema.sql`): Zwei neue Spalten zur event_bookings-Tabelle hinzufuegen:
   - `opt_out_reason TEXT` (nullable)
   - `opt_out_date TIMESTAMP` (nullable)
   WICHTIG: Der CHECK-Constraint auf event_bookings.status enthaelt bereits 'opted_out' — NICHT aendern.

2. **Opt-out Endpoint** (`backend/routes/konfi.js`): Neuen `router.post('/events/:id/opt-out', verifyTokenRBAC, ...)` nach dem bestehenden DELETE-Endpoint (ca. Zeile 1850) hinzufuegen:
   - `reason` aus `req.body` extrahieren
   - Validierung: `reason` muss existieren und `reason.trim().length >= 5` (sonst 400: "Begruendung muss mindestens 5 Zeichen haben")
   - Event laden: `SELECT name, event_date, mandatory, organization_id FROM events WHERE id = $1 AND organization_id = $2`
   - Guard: Event muss existieren UND `mandatory = true` (sonst 400: "Opt-out nur bei Pflicht-Events moeglich")
   - Guard: `event_date > NOW()` (sonst 400: "Event ist bereits vorbei")
   - Status-Wechsel: `UPDATE event_bookings SET status = 'opted_out', opt_out_reason = $3, opt_out_date = NOW() WHERE user_id = $1 AND event_id = $2 AND status = 'confirmed'`
   - Wenn rowCount === 0: 400 "Keine aktive Anmeldung gefunden"
   - Response: `res.json({ message: 'Abmeldung erfolgreich' })`
   - Nach Response (fire-and-forget): Push via PushService.sendEventOptOutToAdmins + liveUpdate falls vorhanden

3. **Opt-in Endpoint** (`backend/routes/konfi.js`): Neuen `router.post('/events/:id/opt-in', verifyTokenRBAC, ...)` direkt danach:
   - Event laden (gleiche Query wie Opt-out) — pruefen ob mandatory und nicht vorbei
   - Status-Wechsel: `UPDATE event_bookings SET status = 'confirmed' WHERE user_id = $1 AND event_id = $2 AND status = 'opted_out'`
   - WICHTIG: opt_out_reason und opt_out_date NICHT loeschen (per User-Decision: "Begruendung bleibt gespeichert")
   - Wenn rowCount === 0: 400 "Keine Opt-out-Anmeldung gefunden"
   - Response: `res.json({ message: 'Wieder angemeldet' })`
   - Nach Response: Push via PushService.sendEventOptInToAdmins + liveUpdate

4. **DELETE-Guard** (`backend/routes/konfi.js`): Im bestehenden `router.delete('/events/:id/register', ...)` (Zeile 1735) am Anfang pruefen:
   - Event laden: `SELECT mandatory FROM events WHERE id = $1`
   - Wenn `mandatory = true`: `return res.status(400).json({ error: 'Pflicht-Events koennen nur ueber Opt-out abgemeldet werden' })`
   - Dies verhindert dass Konfis den DELETE-Endpoint fuer Pflicht-Events missbrauchen

5. **Konfi-Events-Query anpassen** (`backend/routes/konfi.js`): In der GET-Query fuer Konfi-Events (ca. Zeile 1127):
   - `is_registered` anpassen: Von `eb_konfi.id IS NOT NULL` auf `CASE WHEN eb_konfi.status = 'confirmed' THEN true ELSE false END`
   - Neues Feld: `CASE WHEN eb_konfi.status = 'opted_out' THEN true ELSE false END as is_opted_out`
   - `booking_status` wird bereits zurueckgegeben (eb_konfi.status) — opted_out ist automatisch enthalten
