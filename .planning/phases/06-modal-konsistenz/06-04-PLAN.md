  - EventModal (groesstes Formular)

  Fuer jedes dieser Modals:

  1. Eine `isDirty`/`hasUnsavedChanges` State-Variable hinzufuegen (oder berechnen aus formData vs. initialData):
     - Einfachster Ansatz: `const [isDirty, setIsDirty] = useState(false)` und bei jeder Formular-Aenderung `setIsDirty(true)` setzen
     - Bei Submit oder Close mit Erfolg: isDirty zuruecksetzen

  2. canDismiss-Logik in der Parent-Seite (wo useIonModal aufgerufen wird):
     - NICHT in der Modal-Komponente selbst, sondern beim present()-Aufruf
     - Problem: canDismiss braucht Zugriff auf den isDirty-State der Kind-Komponente
     - Loesung: canDismiss als Callback-Prop an die Modal-Komponente uebergeben, die dann entscheidet

  3. Alternativer einfacherer Ansatz (BEVORZUGT):
     - In der Modal-Komponente selbst: handleClose-Funktion erweitern
     - Wenn isDirty: useIonAlert aufrufen mit "Abbrechen" / "Verwerfen"
     - "Abbrechen" schliesst den Alert, Modal bleibt offen
     - "Verwerfen" schliesst Alert UND Modal (via dismiss/onClose)
     - Swipe-to-Dismiss: `canDismiss` Prop am useIonModal present()-Aufruf
       - Callback-Variante: `canDismiss: (data, role) => role !== 'gesture' || !isDirtyRef.current`
       - HINWEIS: isDirty muss als Ref genutzt werden (useRef), da die canDismiss-Closure beim present() erstellt wird

  **Implementierungsdetail:**
  ```typescript
  // In Modal-Komponente:
  const [isDirty, setIsDirty] = useState(false);
  const isDirtyRef = useRef(false);

  // isDirtyRef synchron halten:
  useEffect(() => { isDirtyRef.current = isDirty; }, [isDirty]);

  // Bei Formular-Aenderung:
  const handleFormChange = (field: string, value: any) => {
    setFormData(prev => ({ ...prev, [field]: value }));
    setIsDirty(true);
  };

  // handleClose mit Schutz:
  const [presentAlert] = useIonAlert();

  const handleClose = () => {
    if (isDirty) {
      presentAlert({
        header: 'Ungespeicherte Aenderungen',
        message: 'Moechtest du die Aenderungen verwerfen?',
        buttons: [
          { text: 'Abbrechen', role: 'cancel' },
          { text: 'Verwerfen', role: 'destructive', handler: () => { if (dismiss) dismiss(); else onClose(); } }
        ]
      });
    } else {
      if (dismiss) dismiss(); else onClose();
    }
  };
  ```

  **C) Beim present()-Aufruf in Parent-Seiten canDismiss hinzufuegen:**
  ```typescript
  presentModal({
    presentingElement: presentingElement,

  });
  ```

  Einfachste Loesung: In den Parent-Seiten bei present() KEIN canDismiss setzen.
