  <name>Task 5: useOfflineQuery Stale-Closure Fix (CLN-02)</name>
  <files>frontend/src/hooks/useOfflineQuery.ts</files>
  <action>
    Das Problem: `revalidate` (useCallback Zeile 55) hat `data` in seiner Dependency-Liste (Zeile 84: `[cacheKey, fetcher, ttl, data]`). Dadurch aendert sich die `revalidate`-Referenz bei jedem `data`-Update. Wuerden wir `revalidate` in die useEffect-Dependencies aufnehmen, gaebs einen Loop: data-Update → neue revalidate-Referenz → useEffect re-runs → revalidate() → data-Update → ...

    `data` wird in revalidate nur im catch-Block benoetigt (`if (data !== null)`). Das ist der Stale-Closure.

    Fix-Strategie: `data` aus den `useCallback`-Dependencies entfernen. Im catch-Block statt `data !== null` das vorhandene `setIsStale`-Pattern verwenden (da `setIsStale` einen Funktional-Update vertraegt):

    ```ts
    // VORHER (Zeile 76-81 im catch-Block):
    if (data !== null) {
      setIsStale(true);
    } else {
      setError(message);
      setLoading(false);
    }

    // NACHHER:
    setData((prev) => {
      if (prev !== null) {
        setIsStale(true);
        return prev; // behalte vorhandene Daten
      } else {
        setError(message);
        setLoading(false);
        return null;
      }
    });
    ```

    Dann die useCallback-Dependency (Zeile 84) anpassen:
    ```ts
    // VORHER:
    }, [cacheKey, fetcher, ttl, data]);

    // NACHHER:
    }, [cacheKey, fetcher, ttl]);
    ```

    Danach kann der Kommentar in Zeile 135 aktualisiert werden:
    ```ts
    // VORHER:
    }, [cacheKey, enabled]); // revalidate bewusst nicht in deps (wuerde Loop verursachen)

    // NACHHER:
    }, [cacheKey, enabled, revalidate]);
    ```

    WICHTIG: `setError` und `setLoading` muessen im Funktional-Update per Ref oder direkt aufgerufen werden. Da React setState-Aufrufe ausserhalb eines funktionalen State-Updates ausfuehren kann, ist folgende Alternative sauberer -- `data`-Lesen per Ref:

    Stattdessen: Fuege `dataRef` hinzu der den aktuellen data-Wert trackt:
    ```ts
    const dataRef = useRef<T | null>(null);
    ```

    Nach jedem `setData(transformed)` call auch `dataRef.current = transformed` setzen (und bei `setData(null)` auch `dataRef.current = null`).

    Dann im catch-Block `data` durch `dataRef.current` ersetzen:
    ```ts
    if (dataRef.current !== null) {
      setIsStale(true);
    } else {
      setError(message);
      setLoading(false);
    }
    ```

    Und useCallback ohne `data` in Dependencies:
    ```ts
    }, [cacheKey, fetcher, ttl]);
    ```

    useEffect mit revalidate in Dependencies:
    ```ts
    }, [cacheKey, enabled, revalidate]);
    ```

    Alle bestehenden `setData(transformed)` Aufrufe in revalidate und loadFromCache muessen zusaetzlich `dataRef.current = transformed` setzen. Alle `setData(null)` muessen auch `dataRef.current = null` setzen.

    Vorgehen im Detail:
    1. `const dataRef = useRef<T | null>(null);` nach `const [data, setData] = useState` einfuegen
    2. Suche alle `setData(` Aufrufe in der Datei
    3. Nach jedem `setData(X)` eine Zeile `dataRef.current = X;` einfuegen (oder wenn X = transformed: `dataRef.current = transformed`)
    4. Im catch-Block von revalidate: `data` → `dataRef.current`
    5. useCallback-Dep: `data` entfernen
    6. Initial-useEffect-Dep: `revalidate` hinzufuegen, Kommentar entfernen
  </action>
  <verify>
    <automated>grep -n "data\]" /Users/simonluthe/Documents/Konfipoints/frontend/src/hooks/useOfflineQuery.ts && echo "FEHLER: data noch in deps" || echo "OK: data nicht in deps"</automated>
  </verify>
  <done>revalidate hat kein `data` in seinen useCallback-Dependencies. Der Initial-useEffect hat `revalidate` in seinen Dependencies ohne den alten Kommentar. Ein `dataRef` trackt den aktuellen data-Wert fuer den catch-Block.</done>
