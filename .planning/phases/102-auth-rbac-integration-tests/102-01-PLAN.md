**describe('POST /api/auth/login'):**
- Login mit gueltigem Konfi (per D-05): POST mit { username: USERS.konfi1.username, password: PASSWORD }
  - Expect: 200, body.token definiert (String), body.refresh_token definiert (String), body.user.id === USERS.konfi1.id, body.user.type === 'konfi', body.user.display_name, body.user.organization, body.user.gottesdienst_points definiert
- Login mit gueltigem Admin: POST mit { username: USERS.admin1.username, password: PASSWORD }
  - Expect: 200, body.user.type === 'admin', body.user.role_name === 'admin', KEIN jahrgang/gottesdienst_points im Response
- Login mit falschem Passwort (per D-05): POST mit { username: USERS.konfi1.username, password: 'falsch' }
  - Expect: 401, body.error definiert
- Login mit nicht-existentem User (per D-05): POST mit { username: 'gibtesnicht', password: 'egal' }
  - Expect: 401, body.error definiert
- Login ohne Body: POST mit {}
  - Expect: 400 oder 422 (Validation Error)

**describe('POST /api/auth/refresh') (per D-06):**
- Refresh mit gueltigem Token: Zuerst Login (POST /login), dann POST /refresh mit dem refresh_token aus Login-Response
  - Expect: 200, body.token (neuer Access-Token), body.refresh_token (neuer Refresh-Token, NICHT gleich dem alten)
- Refresh mit bereits verwendetem Token (Rotation): Login, Refresh (1x), nochmal Refresh mit ALTEM Token
  - Expect: 401 (Token wurde rotiert/revoked)
- Refresh ohne Token: POST /refresh mit {}
  - Expect: 400
- Refresh mit ungueltigem Token: POST /refresh mit { refresh_token: 'gibts-nicht' }
  - Expect: 401

**describe('POST /api/auth/logout') (per D-07):**
- Logout mit Token-Revoke: Login, dann POST /logout mit Bearer-Token UND { refresh_token } im Body
  - Expect: 200, body.message === 'Logout erfolgreich'
  - Danach: POST /refresh mit dem alten refresh_token sollte 401 geben
- Logout ohne refresh_token im Body: POST /logout mit Bearer-Token aber leerem Body
  - Expect: 200 (best-effort Logout)
- Logout ohne Auth-Header: POST /logout ohne Bearer-Token
  - Expect: 401

**describe('GET /api/auth/me'):**
- Profil mit gueltigem Token: generateToken('konfi1'), GET /me mit Bearer
  - Expect: 200, body.id === USERS.konfi1.id, body.username, body.display_name, body.role_name
- Profil ohne Token: GET /me ohne Header
  - Expect: 401

**describe('POST /api/auth/change-password'):**
- Passwort aendern mit korrektem altem Passwort: generateToken('konfi1'), POST mit { currentPassword: PASSWORD, newPassword: 'neuesPasswort123' }
  - Expect: 200
  - Danach: Login mit neuem Passwort muss 200 geben
- Passwort aendern mit falschem altem Passwort: POST mit { currentPassword: 'falsch', newPassword: 'neuesPasswort123' }
  - Expect: 400
- Passwort zu kurz: POST mit { currentPassword: PASSWORD, newPassword: 'kurz' }
  - Expect: 400

**describe('POST /api/auth/register-konfi'):**
- Registrierung mit gueltigem Invite-Code:
  1. Zuerst: generateToken('orgAdmin1'), POST /api/auth/invite-code mit { jahrgang_id: JAHRGAENGE.jahrgang1.id } -> invite_code aus Response
  2. Dann: POST /api/auth/register-konfi mit { invite_code, display_name: 'Neuer Konfi', username: 'neukonfi', password: 'testPasswort123' }
  - Expect: 200 oder 201, body.token definiert, body.user definiert
- Registrierung mit ungueltigem Invite-Code: POST mit { invite_code: 'INVALID', ... }
  - Expect: 400 oder 404
