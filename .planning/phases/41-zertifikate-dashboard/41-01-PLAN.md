<task type="auto">
  <name>Task 1: DB-Migration + Zertifikat-CRUD in teamer.js</name>
  <files>backend/routes/teamer.js</files>
  <action>
1. Idempotente Migration am Anfang von teamer.js (gleiches Pattern wie badges.js):
   - `certificate_types` Tabelle: id SERIAL PK, name VARCHAR(100) NOT NULL, icon VARCHAR(50) DEFAULT 'ribbon', organization_id INT REFERENCES organizations(id), is_active BOOLEAN DEFAULT true, created_at TIMESTAMP DEFAULT NOW(). UNIQUE(organization_id, name).
   - `user_certificates` Tabelle: id SERIAL PK, user_id INT REFERENCES users(id), certificate_type_id INT REFERENCES certificate_types(id), organization_id INT REFERENCES organizations(id), issued_date DATE NOT NULL, expiry_date DATE NULL, admin_id INT REFERENCES users(id), created_at TIMESTAMP DEFAULT NOW(). UNIQUE(user_id, certificate_type_id).

2. CRUD-Endpoints fuer Zertifikat-Typen (Admin-only, requireOrgAdmin):
   - GET /teamer/certificate-types: Alle Typen der Organisation (is_active=true), sortiert nach name
   - POST /teamer/certificate-types: Neuen Typ erstellen (name, icon), Validierung name nicht leer
   - PUT /teamer/certificate-types/:id: Typ bearbeiten (name, icon, is_active)
   - DELETE /teamer/certificate-types/:id: Typ loeschen (nur wenn keine user_certificates zugewiesen, sonst 409)

3. Zertifikat-Zuweisung an Teamer (Admin-only):
   - POST /teamer/:userId/certificates: Zertifikat zuweisen (certificate_type_id, issued_date, expiry_date optional). Pruefen: User existiert, ist Teamer, Typ gehoert zur Organisation.
   - DELETE /teamer/:userId/certificates/:certId: Zertifikat entfernen
   - GET /teamer/:userId/certificates: Alle Zertifikate eines Teamers (fuer Admin-Detail-View)

4. GET /teamer/dashboard: Neuer Endpoint (requireTeamer), liefert:
   - greeting: { display_name, hour: aktuelle Stunde (Server-Zeit) } -- Frontend bestimmt Begruessungstext
   - certificates: Alle certificate_types der Org mit LEFT JOIN user_certificates fuer den aktuellen Teamer. Felder: id, name, icon, issued_date (null wenn nicht erhalten), expiry_date, status ('valid'|'expired'|'not_earned')
   - events: Naechste 3 anstehende Events (Teamer-Events + Teamer-gesucht-Events), gleiche Query wie in events.js aber LIMIT 3, event_date >= NOW()
   - badges: { recent: letzte 3 earned Badges (icon, name, earned_at), earned_count, total_count }
   - config: Dashboard-Config aus settings (teamer_dashboard_show_zertifikate, teamer_dashboard_show_events, teamer_dashboard_show_badges, teamer_dashboard_show_losung). Defaults: alle true.

Events-Query fuer Dashboard: Events mit target_role IN ('teamer','all') ODER events mit needs_teamer=true, event_date >= CURRENT_DATE, organization_id = user.org, ORDER BY event_date ASC LIMIT 3. Join mit event_bookings fuer is_registered Status.

     FROM user_certificates uc
