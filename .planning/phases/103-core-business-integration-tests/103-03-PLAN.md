    Erstelle `backend/tests/routes/badges.test.js` mit folgenden describe-Bloecken (per D-13, D-14):

    **Standard-Boilerplate.**

    **1. GET /api/admin/badges (requireTeamer):**
    - Admin bekommt 200 + Array mit Badges der eigenen Org (4 Badges im Seed)
    - Teamer bekommt 200 (requireTeamer erlaubt)
    - Konfi bekommt 403
    - Admin2 (Org 2) sieht nur Org-2-Badges (1 Badge: streak2)

    **2. GET /api/admin/badges/:id (requireTeamer):**
    - Admin bekommt 200 + einzelnes Badge mit Details
    - Nicht-existierende ID -> 404
    - Badge aus anderer Org -> 404

    **3. GET /api/admin/badges/criteria-types (requireTeamer):**
    - Bekommt 200 + alle verfuegbaren criteria_types als Objekt

    **4. POST /api/admin/badges (requireAdmin, per D-13):**
    - Admin erstellt neues Badge: { name: 'Test-Badge', criteria_type: 'total_points', criteria_value: 10, icon: 'star', color: '#ff0000' } -> 201
    - Pruefe Response enthaelt id, name, criteria_type
    - Teamer bekommt 403 (nur requireAdmin)
    - Ungueltiger criteria_type -> 400

    **5. PUT /api/admin/badges/:id (requireAdmin):**
    - Admin aktualisiert Badge (id=1) -> 200
    - Badge aus anderer Org -> 404

    **6. DELETE /api/admin/badges/:id (requireAdmin, per D-13):**
    - Admin loescht Badge -> 200
    - Vor dem Loeschen: user_badges Eintraege fuer dieses Badge werden mitgeloescht (CASCADE oder explizit)
    - Badge aus anderer Org -> 404

    **7. Level-Zuordnung (per D-13):**
    - Konfi1 hat 0 Punkte im Seed -> Level "Novize" (points_required: 0)
    - Aktualisiere konfi_profiles Punkte auf 10 direkt in DB (UPDATE)
    - Pruefe ueber GET /api/konfi/dashboard (mit konfi1 Token) dass Level "Gehilfe" angezeigt wird
    - HINWEIS: Level-Zuordnung geschieht on-the-fly bei Dashboard-Abfrage, nicht bei Punkte-Update

    **8. Auto-Award-Trigger (per D-14):**
    - checkAndAwardBadges wird nach Activity-Zuweisung aufgerufen
    - Strategie: Weise Activity mehrfach zu (POST /api/admin/activities/assign-activity mit admin1 Token)
    - Fuer streak-Badge (criteria_value=3): 3x verschiedene Tage zuweisen
    - Danach pruefe: SELECT FROM konfi_badges WHERE konfi_id=1 AND badge_id=1
    - ALTERNATIV: Wenn assign-activity checkAndAwardBadges intern aufruft, reicht das
    - Falls checkAndAwardBadges nicht automatisch laeuft: manuell per DB-Query testen

    **9. Progress-Berechnung (per D-14):**
    - GET /api/konfi/badges (mit konfi1 Token) liefert Badges MIT progress-Feld
    - Pruefe dass progress fuer total_points Badge korrekt berechnet wird (aktuelle Punkte / criteria_value)
    - Pruefe dass progress zwischen 0 und 1 (oder 0-100%) liegt

    Falls globalSetup.js fehlende Tabellen/Spalten braucht (user_badges, konfi_badges, badge_progress), diese ergaenzen.

    WICHTIG: badges.js definiert checkAndAwardBadges als interne Funktion (Zeile 1-666 ca.). Diese wird von activities.js aufgerufen. Der Auto-Award-Test kann entweder ueber den Activities-Endpoint (POST /api/admin/activities/assign-activity) oder direkt per DB-Manipulation + manuellem Trigger erfolgen.
