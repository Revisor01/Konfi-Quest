- Events mit `e.cancelled = true` NUR inkludieren wenn `eb_konfi.id IS NOT NULL` (Konfi hat Booking)
- Am einfachsten als WHERE-Clause: `AND (e.cancelled IS NOT TRUE OR eb_konfi.id IS NOT NULL)`

c) Die Query-Parameter anpassen: `[req.user.organization_id, konfiId, jahrgangId]`

**2. konfi-managment.js — Auto-Enrollment bei POST (EVT-v19-04):**

```javascript
// Auto-Enrollment fuer zukuenftige Pflicht-Events
try {
  const enrollFutureEventsQuery = `
    INSERT INTO event_bookings (event_id, user_id, status, booking_date, organization_id)
    SELECT e.id, $1, 'confirmed', NOW(), $2
    FROM events e
    JOIN event_jahrgang_assignments eja ON e.id = eja.event_id
    WHERE eja.jahrgang_id = $3
      AND e.mandatory = true
      AND e.event_date > NOW()
      AND e.organization_id = $2
      AND e.cancelled IS NOT TRUE
    ON CONFLICT (user_id, event_id) DO NOTHING
  `;
  await db.query(enrollFutureEventsQuery, [userId, req.user.organization_id, jahrgang_id]);
} catch (enrollErr) {
  console.error('Auto-enrollment fuer Pflicht-Events fehlgeschlagen:', enrollErr);
}
```
Diesen Block NACH dem COMMIT aber VOR dem `res.status(201).json(...)` einfuegen (identisch zum Pattern in auth.js Zeile 614-626).

**1. KonfiEventsPage.tsx — Segment-Reihenfolge und Filter-Logik (EVT-v19-08):**

a) `activeTab` State-Typ aendern von `'upcoming' | 'registered' | 'konfirmation'` zu `'meine' | 'alle' | 'konfirmation'`.
   Default-Wert: `'meine'` (statt `'upcoming'`).

b) `getFilteredEvents()` anpassen:
```typescript
switch (activeTab) {
  case 'meine':
    // Persoenliche Event-Historie: alle Events wo angemeldet (inkl. vergangene, abgesagte, opted_out)
    filteredEvents = events.filter(event =>
      event.is_registered || event.booking_status === 'opted_out'
    );
    break;
  case 'alle':
    // NUR zukuenftige Events (keine vergangenen), keine Konfirmation
    filteredEvents = events.filter(event =>
      new Date(event.event_date) >= now &&
      !event.category_names?.toLowerCase().includes('konfirmation')
    );
    break;
  case 'konfirmation':
    filteredEvents = events.filter(event =>
      event.category_names?.toLowerCase().includes('konfirmation')
    );
    break;
  default:
    filteredEvents = events;
}
```
