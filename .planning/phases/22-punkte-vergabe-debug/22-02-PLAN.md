<task type="auto">
  <name>Task 2: Points-History Berechnung konsistent machen</name>
  <files>backend/routes/konfi.js, frontend/src/components/konfi/modals/PointsHistoryModal.tsx</files>
  <action>
**Problem:** Backend berechnet `totals.gottesdienst` als `konfi_profiles.gottesdienst_points - eventGD - bonusGD` (also "reine Aktivitaeten-Punkte"), aber das Frontend berechnet eigene Totals durch Summierung aller History-Eintraege (calculateTotals Funktion). Die Backend-Totals werden im Frontend ignoriert.

**Backend-Fix (konfi.js, GET /points-history):**
Die Totals-Berechnung vereinfachen. Statt "reine Aktivitaeten" zu berechnen, die tatsaechlichen konfi_profiles-Werte als Gesamtpunkte senden:

```javascript
const totals = {
  gottesdienst: konfiProfile?.gottesdienst_points || 0,
  gemeinde: konfiProfile?.gemeinde_points || 0,
  total: (konfiProfile?.gottesdienst_points || 0) + (konfiProfile?.gemeinde_points || 0)
};
```

Die alten Felder `bonus` und `event` aus totals entfernen -- sie sind redundant, da Bonus und Event-Punkte bereits in gottesdienst_points/gemeinde_points enthalten sind.

**Frontend-Fix (PointsHistoryModal.tsx):**
1. Die `calculateTotals()` Funktion komplett entfernen (Zeilen ~131-158).
2. Die Variable `calculatedTotals` durch direkte Nutzung von `totals` (aus dem Backend) ersetzen.
3. Im Header die Stats-Boxen anpassen:
   - GESAMT: `totals.total`
   - GOTTESDIENST: `totals.gottesdienst`
   - GEMEINDE: `totals.gemeinde`
4. Die zweite Stats-Reihe (EVENTS, BONUS) durch Zaehlung aus `history` ersetzen:
   - EVENTS: `history.filter(h => h.source_type === 'event').length`
   - BONUS: `history.filter(h => h.source_type === 'bonus').length`
5. Das `PointsTotals` Interface anpassen: nur noch `gottesdienst`, `gemeinde`, `total`.

So kommen die Punkte-Totals immer aus konfi_profiles (Single Source of Truth) und stimmen mit dem Dashboard ueberein.
