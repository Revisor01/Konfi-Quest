<!-- API-Endpoints aus Plan 01 -->

Material-API:
```
GET    /api/material           — Liste (query: tag_id, search, event_id)
GET    /api/material/:id       — Detail mit Dateien und Tags
POST   /api/material           — Erstellen {title, description?, event_id?, jahrgang_id?, tag_ids?}
PUT    /api/material/:id       — Bearbeiten {title?, description?, event_id?, jahrgang_id?, tag_ids?}
DELETE /api/material/:id       — Loeschen

POST   /api/material/:id/files — Dateien hochladen (multipart, field: 'files')
GET    /api/material/files/:fn — Datei herunterladen (MD5-Hash als Filename)
DELETE /api/material/files/:id — Einzelne Datei loeschen

GET    /api/material/tags      — Alle Tags
POST   /api/material/tags      — Tag erstellen {name}
PUT    /api/material/tags/:id  — Tag umbenennen {name}
DELETE /api/material/tags/:id  — Tag loeschen

GET    /api/material/by-event/:eventId — Material zu einem Event
```

Bestehende Patterns:
- useIonModal mit presentingElement, onClose/onSuccess
- IonItemSliding + IonItemOption fuer Swipe-to-Delete
- app-list-item CSS-Klassen, app-corner-badge, app-icon-circle
- Amber/Gold Akzentfarbe: #d97706
- documentIcon Import: `import { document as documentIcon } from 'ionicons/icons'`
- FileOpener Pattern aus ChatRoom.tsx (oder alternativ window.open fuer Download)
