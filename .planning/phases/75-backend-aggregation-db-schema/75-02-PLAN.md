const today = new Date();

Per D-22: Teamer-Wrapped Auto-Trigger:
```javascript
// Am 1. Dezember: Teamer-Wrapped fuer alle Organisationen
if (today.getMonth() === 11 && today.getDate() === 1) {
  const { rows: orgs } = await this.db.query('SELECT id FROM organizations');
  for (const org of orgs) {
    // Pruefe ob schon generiert (DAT-04 / D-24)
    const { rows: existing } = await this.db.query(
      `SELECT 1 FROM wrapped_snapshots WHERE organization_id = $1 AND wrapped_type = 'teamer' AND year = $2 LIMIT 1`,
      [org.id, today.getFullYear()]

    if (existing.length === 0) {
      await wrappedRouter.generateAllTeamerWrapped(this.db, org.id, today.getFullYear());
    }
  }
}
```

**1. konfi.js: GET /dashboard erweitern**

Im bestehenden Dashboard-Endpoint eine zusaetzliche Query hinzufuegen:
```sql
-- Pruefe ob Wrapped fuer diesen Konfi verfuegbar ist
SELECT EXISTS(
  SELECT 1 FROM wrapped_snapshots
  WHERE user_id = $1 AND wrapped_type = 'konfi'
) as has_wrapped
```

ODER alternativ ueber jahrgaenge.wrapped_released_at:
```sql
SELECT EXISTS(
  SELECT 1 FROM jahrgaenge j
  JOIN konfi_profiles kp ON kp.jahrgang_id = j.id
  WHERE kp.user_id = $1
    AND j.wrapped_released_at IS NOT NULL
    AND j.wrapped_released_at <= NOW()
) as has_wrapped
```

Verwende die zweite Variante (ueber wrapped_released_at), weil:
- Admin kann Wrapped zurueckziehen (DELETE setzt wrapped_released_at = NULL)
- Zeitsteuerung moeglich (released_at in der Zukunft)
- Kein JOIN auf wrapped_snapshots noetig im heissen Dashboard-Pfad

Fuege `has_wrapped: wrappedResult.rows[0]?.has_wrapped || false` zum Dashboard-Response hinzu.

**2. teamer.js: Dashboard/Profil erweitern**

Analoge Query fuer Teamer:
```sql
SELECT EXISTS(
  SELECT 1 FROM wrapped_snapshots
  WHERE user_id = $1 AND wrapped_type = 'teamer'
) as has_wrapped
```

Teamer hat kein wrapped_released_at auf Jahrgaenge (Teamer-Wrapped ist org-weit). Daher direkt auf wrapped_snapshots pruefen.

Fuege `has_wrapped` zum Teamer-Dashboard/Profil-Response hinzu.
