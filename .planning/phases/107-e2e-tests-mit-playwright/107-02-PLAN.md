**e2e/punkte-vergabe.spec.ts** erstellen (per D-05):

Der Test bildet die User-Journey ab: Admin loggt sich ein, navigiert zur Konfi-Verwaltung, vergibt eine Aktivitaet an konfi1, dann loggt sich konfi1 ein und prueft ob die Punkte sichtbar sind.

```typescript
import { test, expect } from '@playwright/test';
import { loginAs } from './helpers/auth';

test.describe('Punkte-Vergabe', () => {
  test('Admin vergibt Aktivitaet, Konfi sieht Punkte', async ({ page }) => {
    // 1. Als Admin einloggen
    await loginAs(page, 'admin1');

    // 2. Zur Konfi-Verwaltung navigieren
    //    EXECUTOR: Lese Admin-Navigation in App.tsx / AdminTabs.tsx
    //    Typisch: Tab "Konfis" oder "Verwaltung" -> Konfi-Liste -> konfi1 auswaehlen
    await page.goto('/app/admin/konfis');  // Anpassen an echte Route
    await page.locator('ion-item', { hasText: /Test Konfi 1|konfi1/i }).click();

    // 3. Aktivitaet vergeben
    //    EXECUTOR: Lese KonfiDetail.tsx oder AdminKonfiDetail.tsx
    //    Typisch: Button "Aktivitaet vergeben" -> Modal mit Aktivitaetsliste -> "Sonntagsgottesdienst" waehlen
    await page.getByRole('button', { name: /aktivit/i }).click();
    await page.locator('ion-item, ion-radio', { hasText: /Sonntagsgottesdienst/i }).click();
    // Bestaetigen
    await page.getByRole('button', { name: /speichern|vergeben|best/i }).click();

    // 4. Warten auf Erfolgsmeldung
    await expect(page.locator('ion-toast, .success-message')).toBeVisible({ timeout: 5_000 });

    // 5. Als Konfi einloggen und Punkte pruefen
    await page.goto('/app/login');
    await loginAs(page, 'konfi1');

    // 6. Punkte-Anzeige pruefen (Dashboard oder Profil)
    //    EXECUTOR: Konfi-Dashboard hat typischerweise Punkte-Widget
    //    Pruefe ob "1" (Gottesdienst-Punkt) irgendwo sichtbar ist
    await expect(page.locator('ion-content')).toContainText(/1/);
  });
});
```

**e2e/chat.spec.ts** erstellen (per D-07):

Der Test bildet die User-Journey ab: konfi1 sendet eine Nachricht im Jahrgangs-Chat, dann loggt sich admin1 ein und sieht die Nachricht.

WICHTIG: Chat verwendet Socket.IO fuer Echtzeit. Im E2E-Test muessen wir beruecksichtigen:
- Socket.IO laeuft im Backend-Container — Frontend verbindet sich via WebSocket
- Nach dem Senden einer Nachricht genuegt ein Page-Reload beim Empfaenger (kein Echtzeit noetig)

```typescript
import { test, expect } from '@playwright/test';
import { loginAs } from './helpers/auth';

test.describe('Chat', () => {
  test('Nachricht wird gesendet und beim Empfaenger angezeigt', async ({ browser }) => {
    const uniqueMsg = `E2E-Test-Nachricht ${Date.now()}`;

    // 1. Als konfi1 einloggen und Nachricht senden
    const konfiPage = await browser.newPage();
    await loginAs(konfiPage, 'konfi1');

    // 2. Zum Jahrgangs-Chat navigieren
    //    EXECUTOR: Lese Chat-Navigation — Tab "Chat" -> Chat-Raum-Liste -> "Jahrgang 2025/2026"
    await konfiPage.goto('/app/chat');  // Anpassen an echte Route
    await konfiPage.locator('ion-item, ion-card', { hasText: /Jahrgang 2025/i }).click();

    // 3. Nachricht eingeben und senden
    //    EXECUTOR: Lese ChatRoom.tsx — Input-Feld und Sende-Button
    await konfiPage.locator('ion-textarea textarea, ion-input input, textarea').fill(uniqueMsg);
    await konfiPage.locator('ion-button[type="submit"], button[type="submit"], ion-icon[name*="send"]').click();

    // 4. Nachricht erscheint im eigenen Chat
    await expect(konfiPage.locator('.message, ion-item', { hasText: uniqueMsg })).toBeVisible({ timeout: 5_000 });
    await konfiPage.close();

    // 5. Als admin1 einloggen und Nachricht pruefen
    const adminPage = await browser.newPage();
    await loginAs(adminPage, 'admin1');
    await adminPage.goto('/app/chat');  // Anpassen an echte Route
    await adminPage.locator('ion-item, ion-card', { hasText: /Jahrgang 2025/i }).click();

    // 6. Gesendete Nachricht sichtbar beim Empfaenger
    await expect(adminPage.locator('.message, ion-item', { hasText: uniqueMsg })).toBeVisible({ timeout: 10_000 });
    await adminPage.close();
  });
});
```

- Chat-Routing: Route zur Chat-Liste und zum Chat-Raum
- ChatRoom-Komponente: Input-Feld, Sende-Button, Nachrichten-Darstellung
- Falls Socket.IO im E2E-Container nicht zuverlaessig funktioniert: Nachricht per REST-API senden (Fallback) oder Page-Reload erzwingen
- Selektoren an tatsaechliche UI anpassen — Chat hat oft custom CSS-Klassen statt Ionic-Standardkomponenten
