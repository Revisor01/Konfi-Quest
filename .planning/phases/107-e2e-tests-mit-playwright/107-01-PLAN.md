1. **e2e/helpers/auth.ts** erstellen — wiederverwendbarer Login-Helper (per D-04, D-09):

```typescript
import { Page } from '@playwright/test';

export async function loginAs(page: Page, username: string, password = 'testpasswort123') {
  await page.goto('/app/login');
  await page.getByPlaceholder(/benutzername/i).fill(username);
  await page.getByPlaceholder(/passwort/i).fill(password);
  await page.getByRole('button', { name: /einloggen|anmelden|login/i }).click();
  // Warten bis Login abgeschlossen (URL wechselt weg von /login)
  await page.waitForURL(/\/app\/(?!login)/, { timeout: 10_000 });
}
```

WICHTIG: Selektoren muessen zur tatsaechlichen Login-Seite passen. Die Ionic-App hat moeglicherweise `ion-input` statt native `<input>`. Pruefen:
- `frontend/src/pages/Login.tsx` oder aehnlich lesen
- Korrekte Selektoren verwenden (getByPlaceholder, getByLabel, getByRole oder data-testid)
- Falls noetig, data-testid Attribute zur Login-Seite hinzufuegen fuer stabile Selektoren

2. **e2e/login.spec.ts** erstellen (per D-04):

```typescript
import { test, expect } from '@playwright/test';
import { loginAs } from './helpers/auth';

test.describe('Login-Flow', () => {
  test('Konfi kann sich einloggen und sieht Dashboard', async ({ page }) => {
    await loginAs(page, 'konfi1');
    // Konfi-Dashboard pruefen: URL enthaelt /app/ und typische Dashboard-Elemente sichtbar
    await expect(page).toHaveURL(/\/app\//);
    // Konfi sieht seinen Namen oder Dashboard-Titel
    await expect(page.locator('ion-content')).toContainText(/dashboard|konfi|punkte/i);
  });

  test('Admin kann sich einloggen und sieht Dashboard', async ({ page }) => {
    await loginAs(page, 'admin1');
    await expect(page).toHaveURL(/\/app\//);
    // Admin sieht Admin-spezifische Elemente
    await expect(page.locator('ion-content')).toContainText(/dashboard|admin|verwaltung/i);
  });

  test('Falsches Passwort zeigt Fehlermeldung', async ({ page }) => {
    await page.goto('/app/login');
    await page.getByPlaceholder(/benutzername/i).fill('konfi1');
    await page.getByPlaceholder(/passwort/i).fill('falschespasswort');
    await page.getByRole('button', { name: /einloggen|anmelden|login/i }).click();
    // Fehlermeldung sichtbar (Toast oder inline)
    await expect(page.locator('ion-toast, .error-message, [role="alert"]')).toBeVisible({ timeout: 5_000 });
  });
});
```

WICHTIG: Die genauen Selektoren und URLs haengen von der tatsaechlichen App-Struktur ab. Der Executor MUSS:
- `frontend/src/pages/Login.tsx` (oder LoginPage.tsx) lesen um die korrekten Input-Selektoren zu bestimmen
- `frontend/src/App.tsx` lesen um die Dashboard-Route zu kennen
- Falls Ionic ion-input Placeholder nicht mit getByPlaceholder funktionieren: auf `page.locator('ion-input[placeholder*="..."] input')` wechseln
- Falls noetig, minimale data-testid Attribute hinzufuegen (nur login-username, login-password, login-submit)
