Datenmodell + Backend-Vertrag fuer die Jahrgang-Steuerzentrale: jahrgaenge bekommt das Boolean-Flag konfspruch_enabled (D-01), und jahrgaenge.js akzeptiert es in POST/PUT/GET. WICHTIG (Prod-vs-Test-Schema-Falle): Migration 082 (082_self_auto_deletion.sql:24) setzt `ALTER TABLE jahrgaenge ALTER COLUMN confirmation_date SET NOT NULL`. Das Test-Schema (backend/tests/globalSetup.js:149) legt die Spalte dagegen NULLABLE an (`ADD COLUMN IF NOT EXISTS confirmation_date DATE` ohne NOT NULL). Wuerde 119-01 nur die INSERT-Spaltenliste aendern, blieben CI gruen, aber jedes POST /admin/jahrgaenge in Prod schluege mit NOT-NULL-Violation fehl. Deshalb MUSS Migration 094 den Constraint droppen.
