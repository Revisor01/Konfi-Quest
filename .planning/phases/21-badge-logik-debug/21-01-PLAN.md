Zwei Bugs in checkAndAwardBadges fixen:

**Bug 1: Streak-Berechnung bei Jahreswechsel (Zeile 249)**
Aktueller Code setzt `expectedWeek = 52` wenn `expectedWeek === 0`. Das ist falsch, weil manche Jahre ISO-Woche 53 haben.

Fix: Statt hardcoded `52` die tatsaechliche letzte Woche des Vorjahres berechnen. Dafuer eine Hilfsfunktion `getISOWeeksInYear(year)` erstellen, die berechnet ob ein Jahr 52 oder 53 ISO-Wochen hat:
```javascript
function getISOWeeksInYear(year) {
  const dec28 = new Date(Date.UTC(year, 11, 28));
  const dayOfYear = Math.ceil((dec28 - new Date(Date.UTC(year, 0, 1))) / 86400000) + 1;
  return Math.ceil((dayOfYear - (dec28.getUTCDay() || 7) + 10) / 7);
}
```
Dann in der Streak-Logik:
```javascript
if (expectedWeek === 0) {
  expectedYear -= 1;
  expectedWeek = getISOWeeksInYear(expectedYear);
}
```

**Bug 2: activity_combination ignoriert criteria_value (Zeile 143)**
Aktueller Code: `criteria.required_activities.every(req => completedActivities.includes(req))` -- das prueft ob ALLE Aktivitaeten aus der Liste absolviert wurden, ignoriert aber `criteria_value`.

Laut Badge-Beschreibung ("Wert gibt die Mindestanzahl an benoetigten Aktivitaeten aus der Liste an") soll criteria_value die Mindestanzahl sein. Fix:
```javascript
const matchCount = criteria.required_activities.filter(req => completedActivities.includes(req)).length;
earned = matchCount >= badge.criteria_value;
```

Beide Fixes sind innerhalb des switch-Blocks in checkAndAwardBadges.
