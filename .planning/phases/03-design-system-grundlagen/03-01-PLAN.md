```typescript
interface SectionHeaderProps {
  title: string;
  subtitle: string; // REQUIRED per user decision

  preset?: 'events' | 'activities' | 'konfis' | 'users' | 'organizations' | 'badges' | 'requests' | 'jahrgang';
  colors?: { primary: string; secondary: string }; // Fallback wenn kein Preset
  stats: Array<{ value: number; label: string }>; // IMMER genau 3 Werte per user decision
}
```

**3. ListSection.tsx:**

Props-Interface:
```typescript
interface ListSectionProps {
  icon: string; // ionicons icon fuer section header
  title: string;
  count: number;
  iconColorClass?: string; // z.B. 'events', 'activities' -- wird zu app-section-icon--{value}
  children: React.ReactNode; // Die List-Items (bleiben in der View)
  // EmptyState Integration:
  emptyIcon?: string;
  emptyTitle?: string;
  emptyMessage?: string;
  emptyIconColor?: string;
  isEmpty?: boolean; // Wenn true, zeigt EmptyState statt children. Falls nicht angegeben: children werden gerendert
}
```

Rendering:
- IonList mit `inset={true}` und `style={{ margin: '16px' }}`
- IonListHeader mit app-section-icon div (Klasse `app-section-icon app-section-icon--{iconColorClass}`) + IonIcon + IonLabel `{title} ({count})`
- IonCard mit Klasse `app-card`
- IonCardContent
- Wenn `isEmpty` true ist (oder count === 0 und emptyIcon angegeben): EmptyState rendern mit emptyIcon, emptyTitle, emptyMessage, emptyIconColor
- Sonst: children rendern (eine IonList mit lines="none" und transparentem Background)
- Imports: IonList, IonListHeader, IonCard, IonCardContent, IonLabel, IonIcon aus @ionic/react, EmptyState aus ./EmptyState
