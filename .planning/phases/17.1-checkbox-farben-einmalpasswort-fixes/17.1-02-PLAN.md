## Kontext

Zwei Probleme mit der Einmalpasswort-Anzeige:

1. **AdminKonfisPage.tsx (handleAddKonfi)**: Nach Konfi-Erstellung sendet das Backend `temporaryPassword` zurueck, aber das Frontend zeigt es NICHT an -- es wird nur eine Erfolgsmeldung gezeigt. Das Passwort geht verloren.

2. **KonfiDetailView.tsx (handlePasswordReset)**: Bei Passwort-Regenerierung wird `presentAlert` mit HTML in der `message` verwendet (`<br/>`, `<strong>`). Ionic's `IonAlert` rendert standardmaessig KEIN HTML in der message-Property -- der HTML-Code wird als Text angezeigt oder gar nicht sichtbar.

In `frontend/src/components/admin/pages/AdminKonfisPage.tsx`, Funktion `handleAddKonfi` (ca. Zeile 157-176):

Nach dem erfolgreichen API-Call muss das `temporaryPassword` aus der Response in einem Alert angezeigt werden.

1. Stelle sicher, dass `useIonAlert` importiert und initialisiert ist (pruefe ob bereits vorhanden)
2. Aendere `handleAddKonfi`:

```typescript
const handleAddKonfi = async (konfiData: any) => {
    try {
      const response = await api.post('/admin/konfis', konfiData);

      if (konfiData.jahrgang_id) {
        await createOrJoinJahrgangChat(konfiData.jahrgang_id, response.data.id);
      }

      const tempPassword = response.data.temporaryPassword;
      if (tempPassword) {
        presentAlert({
          header: 'Konfi erstellt',
          subHeader: `${response.data.name || konfiData.display_name}`,
          message: `Einmalpasswort: ${tempPassword}`,
          buttons: [
            {
              text: 'Kopieren',
              handler: () => {
                navigator.clipboard.writeText(tempPassword);
                setSuccess('Passwort kopiert');
                return false;
              }
            },
            { text: 'Fertig', role: 'cancel' }
          ]
        });
      } else {
        setSuccess(`Konfi "${response.data.name}" erfolgreich hinzugefĂĽgt`);
      }

      await loadData();
    } catch (err: any) {
      // bestehende Fehlerbehandlung beibehalten
    }
};
```

<task id="17.1-02-2">
<title>KonfiDetailView: Einmalpasswort-Alert ohne HTML-Tags</title>
<instructions>
In `frontend/src/components/admin/views/KonfiDetailView.tsx`, Funktion `handlePasswordReset` (ca. Zeile 332-355):

Das Problem: `message` mit HTML-Tags (`<br/>`, `<strong>`) wird in Ionic Alert nicht korrekt gerendert.

Loesung: Verwende `subHeader` und `message` getrennt, ohne HTML:

```typescript
const handlePasswordReset = async () => {
    try {
      const response = await api.post(`/admin/konfis/${konfiId}/regenerate-password`);
      const tempPassword = response.data.temporaryPassword;
      presentAlert({
        header: 'Einmalpasswort erstellt',
        subHeader: tempPassword,
        message: 'Kopiere das Passwort und gib es dem Konfi weiter.',
        buttons: [
          {
            text: 'Kopieren',
            handler: () => {
              navigator.clipboard.writeText(tempPassword);
              setSuccess('Passwort kopiert');
              return false;
            }
          },
          { text: 'Fertig', role: 'cancel' }
        ]
      });
      window.dispatchEvent(new CustomEvent('konfis-updated'));
    } catch (err) {
      setError('Fehler beim ZurĂĽcksetzen des Passworts');
    }
};
```

Die Aenderung: `subHeader` zeigt das Passwort prominent an (Ionic rendert das als groesseren Text unter dem Header), `message` enthaelt den Hinweistext ohne HTML.
</instructions>
