Neuen Endpoint in konfi-managment.js hinzufuegen (vor dem module.exports bzw. return router):

POST /admin/konfis/:id/promote-teamer
- Middleware: requireAdmin (org_admin + admin duerfen befoerdern)
- Validierung:
  1. User existiert in gleicher Organisation
  2. User hat aktuell role_name = 'konfi' (nicht bereits Teamer/Admin)
- Transition in einer DB-Transaction:
  1. Teamer-Role-ID laden: `SELECT id FROM roles WHERE name = 'teamer' AND organization_id = $1`
     - Falls keine Teamer-Rolle existiert: globale Rolle nutzen: `SELECT id FROM roles WHERE name = 'teamer' AND organization_id IS NULL`
  2. `UPDATE users SET role_id = $1 WHERE id = $2` (Rolle aendern)
  3. `DELETE FROM event_bookings WHERE user_id = $1` (Event-Buchungen loeschen)
  4. `DELETE FROM konfi_activities WHERE konfi_id = $1 AND activity_id IN (SELECT id FROM activities WHERE ... )` -- NUR offene Antraege (activity_requests mit status='pending') loeschen:
     `DELETE FROM activity_requests WHERE konfi_id = $1 AND status = 'pending'`
  5. konfi_profiles-Zeile BLEIBT bestehen (nicht loeschen!)
  6. konfi_badges BLEIBEN bestehen (nicht loeschen!)
  7. Chat-Teilnahmen BLEIBEN bestehen (chat_participants nicht aendern)
- Response: `{ success: true, message: 'Konfi wurde zum Teamer befördert', user: { id, display_name, role_name: 'teamer' } }`
- Fehler: 404 wenn User nicht existiert/nicht Konfi, 403 wenn keine Berechtigung

Beachte das Pattern der bestehenden DELETE-Route (Zeile 344ff) als Referenz fuer Organisation-Check und Client-Transaction.
