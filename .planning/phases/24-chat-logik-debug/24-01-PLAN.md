  <files>backend/routes/chat.js</files>
  <action>
In der Route `GET /files/:filename` (Zeile 933-1003) folgende Aenderungen:

1. **Path-Traversal-Schutz hinzufuegen** -- direkt nach dem Token-Verify-Block (Zeile 948), vor dem DB-Query:
   ```javascript
   const filename = path.basename(req.params.filename);
   ```
   Statt `const filename = req.params.filename;` (Zeile 949). `path.basename()` entfernt alle Directory-Komponenten (`../`, `./`, absolute Pfade).

2. **Zusaetzliche Validierung** -- nach dem basename-Aufruf pruefen, dass der Filename nur erlaubte Zeichen enthaelt (Hex-Hash aus multer):
   ```javascript
   if (!/^[a-f0-9]+$/.test(filename)) {
     return res.status(400).json({ error: 'Ungueltiger Dateiname' });
   }
   ```
   Da multer MD5-Hashes als Dateinamen generiert (server.js Zeile 255), sind nur Hex-Zeichen gueltig.

3. **Content-Disposition Header** -- da die Dateien als Hashes gespeichert werden, den originalen Dateinamen aus der DB fuer den Download-Header verwenden. Nach der membership-Pruefung die Query erweitern um `cm.file_name`:
   ```sql
   SELECT cm.room_id, cm.file_name
   FROM chat_messages cm
   JOIN chat_rooms cr ON cm.room_id = cr.id
   WHERE cm.file_path = $1 AND cr.organization_id = $2
   ```
   Und dann vor `res.sendFile`:
   ```javascript
   if (fileMessage.file_name) {
     const ext = path.extname(fileMessage.file_name).toLowerCase();
     if (contentTypes[ext]) {
       res.setHeader('Content-Type', contentTypes[ext]);
     }
   }
   ```
   Damit wird der Content-Type korrekt aus dem Original-Dateinamen abgeleitet, nicht aus dem Hash.
