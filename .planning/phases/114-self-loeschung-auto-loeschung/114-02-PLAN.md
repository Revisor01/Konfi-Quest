Fuege in auth.js eine neue Route `router.post('/delete-account', rbacVerifier, async (req, res) => {...})`
hinzu (nach change-password einreihen). Logik:
1. `const { password } = req.body;` — fehlt -> 400 "Passwort ist erforderlich".
2. `userId = req.user.id`. User laden (id, password_hash, organization_id). Nicht gefunden -> 404.
3. `bcrypt.compare(password, user.password_hash)` — falsch -> 400 "Aktuelles Passwort ist falsch".
4. Transaktion via `db.getClient()`: BEGIN -> `await deleteKonfiCascade(client, userId, user.organization_id)` -> COMMIT.
   ROLLBACK + 500 bei Fehler (Muster wie konfi-management DELETE).
5. Response 200 `{ message: 'Account erfolgreich geloescht' }`.

Gilt fuer ALLE Rollen (D-01) — kein requireAdmin/requireTeamer, nur rbacVerifier. Da
deleteKonfiCascade alle user-gebundenen Tabellen abdeckt, funktioniert sie auch fuer
Teamer/Admin (konfi_profiles fehlt bei denen ggf. -> DELETE trifft 0 Zeilen, unproblematisch).
Fuege `const { deleteKonfiCascade } = require('../utils/konfiDeletion');` oben hinzu, falls
nicht vorhanden. Deutsche Texte, echte Umlaute, keine Emojis.

Schreibe die Tests in tests/routes/auth.test.js nach bestehendem Muster (seed, generateToken).
