Module-Export: `module.exports = (db, rbacVerifier, roleHelpers, materialUpload) => { ... }`

**Idempotente Migration** (runMaterialMigration, sofort ausgefuehrt):

1. `material_tags` Tabelle:
   - id SERIAL PK, name VARCHAR(100) NOT NULL, organization_id INT REFERENCES organizations(id), created_at TIMESTAMP DEFAULT NOW()
   - UNIQUE(organization_id, name)

2. `materials` Tabelle:
   - id SERIAL PK, title VARCHAR(200) NOT NULL, description TEXT, event_id INT REFERENCES events(id) ON DELETE SET NULL, jahrgang_id INT REFERENCES jahrgaenge(id) ON DELETE SET NULL, organization_id INT REFERENCES organizations(id), created_by INT REFERENCES users(id), created_at TIMESTAMP DEFAULT NOW(), updated_at TIMESTAMP DEFAULT NOW()

3. `material_file_tags` (Join-Tabelle Material <-> Tags):
   - material_id INT REFERENCES materials(id) ON DELETE CASCADE, tag_id INT REFERENCES material_tags(id) ON DELETE CASCADE, PRIMARY KEY(material_id, tag_id)

4. `material_files` Tabelle:
   - id SERIAL PK, material_id INT REFERENCES materials(id) ON DELETE CASCADE, original_name VARCHAR(500) NOT NULL, stored_name VARCHAR(100) NOT NULL, mime_type VARCHAR(100), file_size INT, created_at TIMESTAMP DEFAULT NOW()

**Tag-Endpoints** (requireOrgAdmin):
- GET /tags — Tags der eigenen Organisation laden
- POST /tags — Neuen Tag erstellen (name, organization_id aus req.user)
- PUT /tags/:id — Tag umbenennen
- DELETE /tags/:id — Tag loeschen (CASCADE loescht Zuordnungen)

**Material-Endpoints**:
- GET / — Alle Materialien der Organisation laden (requireTeamer oder requireOrgAdmin). JOIN mit material_tags und material_files (Anzahl). Query-Params: tag_id (Filter), search (ILIKE auf title+description), event_id (Filter). Sortierung: created_at DESC. Ergebnis: Array mit {id, title, description, event_id, event_title, jahrgang_id, jahrgang_name, tags: [{id, name}], file_count, created_by_name, created_at}
- GET /:id — Einzelnes Material mit allen Dateien und Tags (requireTeamer oder requireOrgAdmin). Ergebnis: {id, title, description, event_id, event_title, jahrgang_id, jahrgang_name, tags, files: [{id, original_name, mime_type, file_size, created_at}], created_by_name, created_at}
- POST / — Material erstellen (requireOrgAdmin). Body: {title, description?, event_id?, jahrgang_id?, tag_ids?: number[]}. Setzt organization_id und created_by aus req.user.
- PUT /:id — Material bearbeiten (requireOrgAdmin). Body: {title?, description?, event_id?, jahrgang_id?, tag_ids?: number[]}. Bei tag_ids: DELETE + INSERT in material_file_tags. updated_at = NOW().
- DELETE /:id — Material loeschen (requireOrgAdmin). Dateien vom Dateisystem loeschen (stored_name aus material_files abfragen, dann fs.unlink). CASCADE loescht DB-Eintraege.

**Datei-Endpoints**:
- POST /:id/files — Dateien zu Material hochladen (requireOrgAdmin, materialUpload.array('files', 10)). Fuer jede Datei: INSERT in material_files mit original_name, stored_name (req.file.filename), mime_type, file_size.
- GET /files/:filename — Datei herunterladen (requireTeamer oder requireOrgAdmin). Path-Traversal-Schutz: Nur /^[a-f0-9]{32}$/ als filename akzeptieren. Pruefen ob material_files-Eintrag existiert und Material zur gleichen Organization gehoert. Content-Disposition: attachment mit original_name. Content-Type aus mime_type.
- DELETE /files/:fileId — Einzelne Datei loeschen (requireOrgAdmin). Datei vom Dateisystem loeschen + DB-Eintrag entfernen.

**GET /by-event/:eventId** — Material zu einem bestimmten Event laden (requireTeamer oder requireOrgAdmin). Fuer die Event-Detail-Integration. Gibt Array von Materials zurueck die diesem Event zugeordnet sind.

**Wichtig:**
- organization_id immer aus req.user.organization_id (RBAC)
- requireTeamer erlaubt sowohl Teamer als auch Admins (prueft role >= teamer)
- Alle Queries mit organization_id filtern (Mandantentrennung)
- Echte Umlaute in Fehlermeldungen (nicht ue/ae/oe)
- KEINE Emojis
