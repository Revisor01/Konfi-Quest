**AJG-01 — Datumspicker ohne Endjahreszahlen-Limit**: In AdminJahrgaengeePage.tsx den IonDatetime fuer das Konfirmationsdatum (Zeile 316-328) pruefen. Aktuell hat der IonDatetime kein explizites `max`-Attribut — ABER IonDatetime begrenzt standardmaessig auf das aktuelle Jahr. Um Jahreszahlen bis 2030+ zu ermoeglichen:

Auf dem IonDatetime (id="confirmation-date", Zeile 316-328) das `max`-Attribut hinzufuegen:
```tsx
<IonDatetime
  id="confirmation-date"
  presentation="date"
  locale="de-DE"
  max="2035-12-31"
  value={formData.confirmation_date || undefined}
  onIonChange={(e) => {
    const value = e.detail.value;
    if (typeof value === 'string') {
      setFormData({ ...formData, confirmation_date: value.split('T')[0] });
    }
  }}
  disabled={loading}
/>
```

Das `max="2035-12-31"` erlaubt Konfirmationsdaten bis weit in die Zukunft (2035), was fuer Jahrgaenge ausreichen sollte. Keine Endjahreszahlen-Limitierung auf 2026.
