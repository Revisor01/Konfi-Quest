  const params = timeslotId ? [eventId, timeslotId] : [eventId];
  const { rows: [nextInLine] } = await db.query(query, params);

    **Bug-Erlaeuterung (ABG-02):** `router.push('/admin/chat/123', 'forward')` erzeugt einen schwarzen Screen weil die Navigation von einer Sub-Page (EventDetailView) zu einem anderen Tab (Chat) geht. Das `forward`-Transition ueberlagert sich mit dem Tab-Wechsel.

    **Fix in EventDetailView.tsx:**
    1. Die `handleNavigateToChat`-Funktion (Zeile 397-399) aendern:
       ```tsx
       const handleNavigateToChat = () => {
         if (!eventData?.chat_room_id) return;
         // Erst zurueck zur Events-Liste, dann zum Chat navigieren
         // routerDirection 'root' verhindert schwarzen Screen beim Tab-Wechsel
         router.push(`/admin/chat/${eventData.chat_room_id}`, 'root');
       };
       ```
    2. Die Aenderung von `'forward'` zu `'root'` verhindert die Forward-Transition die beim Tab-Wechsel den schwarzen Screen verursacht. `'root'` ersetzt den aktuellen Stack und navigiert direkt zum Chat-Tab.
