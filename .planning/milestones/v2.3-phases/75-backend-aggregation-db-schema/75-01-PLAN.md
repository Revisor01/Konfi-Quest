```sql
-- Migration: Wrapped Feature
CREATE TABLE IF NOT EXISTS wrapped_snapshots (
  id SERIAL PRIMARY KEY,
  user_id INTEGER NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  organization_id INTEGER NOT NULL REFERENCES organizations(id) ON DELETE CASCADE,
  wrapped_type VARCHAR(10) NOT NULL CHECK (wrapped_type IN ('konfi', 'teamer')),
  jahrgang_id INTEGER REFERENCES jahrgaenge(id) ON DELETE SET NULL,
  year INTEGER NOT NULL,
  data JSONB NOT NULL,
  computed_at TIMESTAMP NOT NULL DEFAULT NOW(),
  UNIQUE(user_id, wrapped_type, year)
);

CREATE INDEX IF NOT EXISTS idx_wrapped_snapshots_org_year
  ON wrapped_snapshots(organization_id, year);
CREATE INDEX IF NOT EXISTS idx_wrapped_snapshots_user
  ON wrapped_snapshots(user_id, wrapped_type);

ALTER TABLE jahrgaenge ADD COLUMN IF NOT EXISTS wrapped_released_at TIMESTAMP;
```

In `backend/routes/wrapped.js` die vollstaendige Implementierung:

**Hilfsfunktion `generateKonfiSnapshot(db, userId, orgId, jahrgangId, year)`:**

Per D-05: Punkte aus konfi_profiles (gottesdienst_points + gemeinde_points).
Per D-06: Events = COUNT aus event_bookings WHERE user_id AND status='confirmed' AND attendance_status='present'. Lieblings-Event = letztes besuchtes Event.
Per D-07: Badges = aus user_badges JOIN custom_badges WHERE user_id AND organization_id.
Per D-08: Chat = COUNT aus chat_messages WHERE user_id AND room in Org-Rooms.
Per D-09: Aktivster Monat = GROUP BY EXTRACT(MONTH FROM completed_date) auf user_activities UNION event_bookings.
Per D-10 / DAT-08: Endspurt = Vergleich (gottesdienst_points + gemeinde_points) vs. (target_gottesdienst + target_gemeinde) aus jahrgaenge. Flag `endspurt: true` + `fehlende_punkte` wenn unter Ziel.

Bonus-Punkte: SUM aus bonus_points WHERE konfi_id = userId (konfi_id Spalte!).

JSONB-Struktur (version 1):

    "badges": { "total_earned": N, "total_available": N, "badges": [{ "name": "...", "icon": "...", "color": "..." }] },

```

**Hilfsfunktion `generateTeamerSnapshot(db, userId, orgId, year)`:**

Per D-11: Events geleitet = COUNT event_bookings WHERE user_id AND role=teamer (JOIN users/roles).
Per D-12: Konfis betreut = COUNT DISTINCT konfi_profiles.user_id in zugewiesenen Jahrgaengen.
Per D-13: Badges = COUNT user_badges WHERE user_id AND organization_id.
Per D-14: Zertifikate = COUNT user_certificates WHERE user_id AND organization_id.
Per D-15: Jahre aktiv = Differenz NOW() - users.teamer_since in Jahren.

JSONB-Struktur (version 1):

    "engagement": { "teamer_seit": "...", "jahre_aktiv": N },
    "zeitraum": { "year": N }
  }

```

**Endpoints implementieren:**

Per D-16: `GET /me` -- rbacVerifier, liest aus wrapped_snapshots WHERE user_id = req.user.id. Typ automatisch erkennen (konfi oder teamer basierend auf req.user.role_name). 404 wenn kein Snapshot existiert.

`GET /status` -- rbacVerifier, prueft ob Wrapped verfuegbar: Konfi = wrapped_released_at auf Jahrgang gesetzt, Teamer = wrapped_snapshots Eintrag existiert. Response: `{ available: boolean, year: number }`.

Per D-17: `POST /generate/:jahrgangId` -- requireAdmin, validiert Jahrgang gehoert zur Org des Admins. Laed alle Konfis des Jahrgangs. Fuer jeden: generateKonfiSnapshot(). UPSERT mit `ON CONFLICT (user_id, wrapped_type, year) DO UPDATE SET data = EXCLUDED.data, computed_at = NOW()`. Setzt jahrgaenge.wrapped_released_at = NOW(). Per D-24: Wenn wrapped_released_at bereits gesetzt, trotzdem neu generieren (Admin-Override).

Per D-18: `POST /generate-teamer` -- requireOrgAdmin, laed alle Teamer der Org (JOIN users/roles WHERE role_name='teamer'). Fuer jeden: generateTeamerSnapshot(). UPSERT analog.

`DELETE /:jahrgangId` -- requireOrgAdmin, loescht wrapped_snapshots WHERE jahrgang_id AND organization_id. Setzt wrapped_released_at = NULL.

**Alle Queries in Transaktion** (db.getClient + BEGIN/COMMIT/ROLLBACK) fuer die Generate-Endpoints.

**Error-Handling:** Konfi mit 0 Aktivitaeten bekommt trotzdem einen Snapshot (alle Werte 0). Kein Abbruch bei einzelnem Fehler -- logge Warning und fahre fort.
