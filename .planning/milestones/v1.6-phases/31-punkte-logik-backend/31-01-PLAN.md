<task type="auto">
  <name>Task 2: Warnung bei Deaktivierung in jahrgaenge.js</name>
  <files>backend/routes/jahrgaenge.js</files>
  <action>
In `backend/routes/jahrgaenge.js` im PUT /:id Handler, VOR dem UPDATE-Query:

1. Bestehende Werte laden: `SELECT gottesdienst_enabled, gemeinde_enabled FROM jahrgaenge WHERE id = $1 AND organization_id = $2`
2. warnings-Array initialisieren
3. Wenn gottesdienst_enabled von true auf false wechselt:
   - Query: `SELECT COUNT(*)::int as count FROM konfi_profiles WHERE jahrgang_id = $1 AND gottesdienst_points > 0`
   - Wenn count > 0: `warnings.push({ type: 'gottesdienst', affected_count: count, message: '${count} Konfi(s) haben bereits Gottesdienst-Punkte' })`
4. Wenn gemeinde_enabled von true auf false wechselt:
   - Query: `SELECT COUNT(*)::int as count FROM konfi_profiles WHERE jahrgang_id = $1 AND gemeinde_points > 0`
   - Wenn count > 0: `warnings.push({ type: 'gemeinde', affected_count: count, message: '${count} Konfi(s) haben bereits Gemeinde-Punkte' })`
5. UPDATE normal ausfuehren (NICHT blockieren!)
6. Response erweitern: `const response = { message: 'Jahrgang erfolgreich aktualisiert' }; if (warnings.length > 0) response.warnings = warnings; res.json(response);`

WICHTIG: Warnung ist NUR Info, KEIN Blocking (kein 409, immer 200 mit optionalem warnings-Array).
