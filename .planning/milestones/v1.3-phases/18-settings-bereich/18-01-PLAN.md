## Goal

Settings-Seite hat korrekte Sektions-Reihenfolge (Konto oben, Verwaltung, Inhalt -- Profil-Block raus), Einladen-Bereich ist mattes Blau, Gottesdienst-Aktivitaeten werden in Blau dargestellt, und Kategorien haben eine eigene Farbe (nicht Orange wie Badges, nicht Gruen wie Aktivitaeten).

## must_haves

- [ ] Profil-Block (Avatar mit Name/Rolle) ist aus der Settings-Seite entfernt (SET-01)
- [ ] Reihenfolge der Sektionen: Konto oben, Verwaltung, Inhalt (SET-01)
- [ ] Einladen-Item (Konfis einladen) verwendet mattes Blau statt Jahrgang-Lila (SET-02)
- [ ] Gottesdienst-Aktivitaeten in der ActivitiesView verwenden Blau statt Gruen (SET-03)
- [ ] Kategorien-Seite verwendet eine eigene Farbe (z.B. Teal/Cyan #06b6d4) statt Gruen (SET-04, SET-05)

**SET-02: Einladen-Item auf mattes Blau**

In `frontend/src/components/admin/pages/AdminSettingsPage.tsx`, finde das Einladen-Item (aktuell `app-list-item--jahrgang` und `app-icon-circle--jahrgang`):

Aendere die CSS-Klassen des Einladen-Items:
- `app-list-item--jahrgang` bleibt (ist schon blau)
- Das ist bereits mattes Blau (#007aff) -- VERIFIZIERE ob der Farbton passt. Falls ja, keine Aenderung noetig.

Falls die Farbe zu kraeftig ist, aendere auf inline style:
```tsx
<div className="app-icon-circle app-icon-circle--lg" style={{ backgroundColor: '#667eea' }}>
```
Und `app-list-item--users` statt `app-list-item--jahrgang` verwenden (mattes Blau #667eea).

**SET-03: Gottesdienst-Aktivitaeten in Blau**

In `frontend/src/components/admin/ActivitiesView.tsx`, aendere die typeColor-Logik (Zeile 187):

Aktuell:
```tsx
const typeColor = '#059669';
```

Aendern zu:
```tsx
const typeColor = activity.type === 'gottesdienst' ? '#007aff' : '#059669';
```

Damit werden Gottesdienst-Aktivitaeten Blau und Gemeinde-Aktivitaeten bleiben Gruen.
