## Goal

AdminBadgesPage hat einen Zurueck-Button oben links und die Badges-Sektionen zeigen Oberkategorie-Icons in ihren Zwischenueberschriften statt des generischen Flash-Icons.

## must_haves

- [ ] AdminBadgesPage hat einen arrowBack-Button oben links im Header (SET-08)
- [ ] Badges-Sektionen Zwischenueberschriften zeigen passende Oberkategorie-Icons statt flash-Icon (SET-09)

## Context

**AdminBadgesPage.tsx** (`frontend/src/components/admin/pages/AdminBadgesPage.tsx`):
- Header (Zeile 150-166): Hat nur `<IonTitle>Badges</IonTitle>` und einen Add-Button rechts
- Es fehlt ein `<IonButtons slot="start">` Block mit arrowBack
- Andere Admin-Seiten wie AdminCategoriesPage und AdminLevelsPage haben den arrowBack-Button bereits implementiert

**BadgesView.tsx** (`frontend/src/components/admin/BadgesView.tsx`):
- Badges werden nach `criteria_type` gruppiert (Zeile 273-278)
- Jede Gruppe hat ein `IonListHeader` mit `app-section-icon--badges` und `flash` Icon (Zeile 287-291)
- Das flash-Icon ist generisch -- jede Oberkategorie sollte ein passendes Icon bekommen
- Verfuegbare criteria_types und passende Icons:
  - `total_points` (Gesamtpunkte) -> `statsChart` oder `trophy`
  - `gottesdienst_points` (Gottesdienst) -> `home`
  - `gemeinde_points` (Gemeinde) -> `people`
  - `specific_activity` (Spezielle Aktivitaet) -> `flash`
  - `both_categories` (Beide Kategorien) -> `grid` oder `layers`
  - `activity_combination` (Aktivitaetskombination) -> `list`
  - `category_activities` (Kategorie-Aktivitaeten) -> `pricetag`
  - `time_based` (Zeitbasiert) -> `time`
  - `activity_count` (Aktivitaetsanzahl) -> `checkmarkCircle`
  - `event_count` (Event-Teilnahmen) -> `calendar`
  - `bonus_points` (Bonuspunkte) -> `star`
  - `streak` (Serie) -> `flame`
  - `unique_activities` (Einzigartige Aktivitaeten) -> `sparkles`

## Tasks

<task id="18-03-1">
<title>SET-08: AdminBadgesPage Zurueck-Button</title>
<instructions>
In `frontend/src/components/admin/pages/AdminBadgesPage.tsx`:

1. **Import hinzufuegen:** Fuege `arrowBack` zu den ionicons-Imports hinzu:

Aktuell:
```tsx
import { add } from 'ionicons/icons';
```

Aendern zu:
```tsx
import { add, arrowBack } from 'ionicons/icons';
```

2. **Zurueck-Button im Header hinzufuegen** (nach `<IonToolbar>`, vor `<IonTitle>`):

Aktuell (ca. Zeile 152-159):
```tsx
<IonToolbar>
  <IonTitle>Badges</IonTitle>
  <IonButtons slot="end">
```

Aendern zu:
```tsx
<IonToolbar>
  <IonButtons slot="start">
    <IonButton onClick={() => window.history.back()}>
      <IonIcon icon={arrowBack} />
    </IonButton>
  </IonButtons>
  <IonTitle>Badges</IonTitle>
  <IonButtons slot="end">
```

**Ergebnis:** AdminBadgesPage hat oben links einen Zurueck-Pfeil wie alle anderen Settings-Unterseiten.
</instructions>
</task>

<task id="18-03-2">
<title>SET-09: Oberkategorie-Icons in Badges-Sektionsueberschriften</title>
<instructions>
In `frontend/src/components/admin/BadgesView.tsx`:

1. **Imports erweitern:** Fuege folgende Icons zu den bestehenden Imports hinzu (pruefe welche schon importiert sind):
- `statsChart` -- wahrscheinlich NICHT importiert, muss hinzugefuegt werden
- `home`, `people`, `pricetag`, `time`, `calendar`, `star`, `flame`, `sparkles`, `checkmarkCircle` -- die meisten davon sind BEREITS importiert (siehe Zeile 17-75)
- `grid` oder `layers` -- muss hinzugefuegt werden fuer `both_categories`
- `list` -- muss hinzugefuegt werden fuer `activity_combination`

Importiere zusaetzlich zu den bestehenden Imports:
```tsx
import {
  // ... bestehende Imports ...,
  statsChart,
  grid,
  listOutline
} from 'ionicons/icons';
```

Pruefe ob `statsChart`, `grid`, `listOutline` NICHT bereits importiert sind, und fuege nur die fehlenden hinzu.

2. **Icon-Mapping-Funktion erstellen:** Fuege nach dem bestehenden `getCriteriaTypeText` (Zeile ~170-187) eine neue Funktion hinzu:

```tsx
const getCriteriaTypeIcon = (type: string) => {
  switch (type) {
    case 'total_points': return statsChart;
    case 'gottesdienst_points': return home;
    case 'gemeinde_points': return people;
    case 'specific_activity': return flash;
    case 'both_categories': return grid;
    case 'activity_combination': return listOutline;
    case 'category_activities': return pricetag;
    case 'time_based': return time;
    case 'activity_count': return checkmarkCircle;
    case 'event_count': return calendar;
    case 'bonus_points': return star;
    case 'streak': return flame;
    case 'unique_activities': return sparkles;
    default: return flash;
  }
};
```

3. **Sektionsueberschriften aktualisieren:** In der Badges-Gruppen-Darstellung (Zeile ~286-291), aendere das generische flash-Icon:

Aktuell:
```tsx
<IonListHeader>
  <div className="app-section-icon app-section-icon--badges">
    <IonIcon icon={flash} />
  </div>
  <IonLabel>{getCriteriaTypeText(criteriaType)} ({typeBadges.length})</IonLabel>
</IonListHeader>
```

Aendern zu:
```tsx
<IonListHeader>
  <div className="app-section-icon app-section-icon--badges">
    <IonIcon icon={getCriteriaTypeIcon(criteriaType)} />
  </div>
  <IonLabel>{getCriteriaTypeText(criteriaType)} ({typeBadges.length})</IonLabel>
</IonListHeader>
```

**Ergebnis:** Jede Badge-Oberkategorie zeigt ein passendes Icon (z.B. Pokal fuer Gesamtpunkte, Haus fuer Gottesdienst, Flamme fuer Serien).
</instructions>
</task>
