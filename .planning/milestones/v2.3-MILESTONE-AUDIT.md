## Kritische Gaps

### 1. teamer_jahrgang_assignments existiert nicht (KRITISCH)

**Datei:** `backend/routes/wrapped.js` Zeile 289
**Problem:** `generateTeamerSnapshot()` referenziert `teamer_jahrgang_assignments tja` -- diese Tabelle existiert nicht in der Datenbank. Das gesamte System verwendet `user_jahrgang_assignments`.

**Auswirkung:**
- Jeder Aufruf von `generateTeamerSnapshot()` gibt PostgreSQL-Fehler
- `POST /api/wrapped/generate-teamer` gibt 500 zurueck
- Cron-Job am 1. Dezember schlaegt fuer alle Organisationen fehl
- `konfis_betreut.total_konfis` und `konfis_betreut.jahrgaenge` immer 0/[]

**Fix:** Zeile 289: `teamer_jahrgang_assignments tja` ersetzen durch `user_jahrgang_assignments uja` (und alle `tja.` durch `uja.`)

**Betroffene Requirements:** DAT-07, TS-01, TS-02, INT-03, INT-04

## Nicht-kritische Issues

### 2. CSS-Klasse wrapped-modal-fullscreen nicht definiert

5 Stellen referenzieren `cssClass: 'wrapped-modal-fullscreen'` in `useIonModal`, aber die Klasse ist nirgends definiert. Funktioniert trotzdem wegen `.wrapped-overlay { position: fixed; inset: 0; z-index: 9999 }`.

**Betroffene Requirements:** UI-02 (minimal)

### 3. useRef stale-props Risiko bei Wiederansicht

`wrappedModalRef` Pattern in ProfileView, TeamerProfilePage, TeamerKonfiStatsPage koennte bei mehreren Snapshots stale Props liefern. Fuer Nutzer mit nur einem Snapshot nicht bemerkbar.

**Betroffene Requirements:** D-01, D-02, D-03 (Risiko, kein harter Bug)

### 4. Orphaned API Endpoint

`GET /api/wrapped/status` implementiert aber ohne Frontend-Consumer.
