Neue Datei `frontend/src/services/offlineCache.ts` erstellen. Per D-01, D-02, D-03, D-04, D-05.

Der Service nutzt Capacitor Preferences (gleiche Dependency wie tokenStore) und speichert Cache-Eintraege als JSON:

```typescript
interface CacheEntry<T> {
  data: T;
  timestamp: number;
  ttl: number;
}
```

Exportiertes API (als `offlineCache` Objekt, KEIN class):

1. `async get<T>(key: string): Promise<CacheEntry<T> | null>` — Liest aus Preferences, parsed JSON. Bei korruptem JSON (parse error) den Key loeschen und null zurueckgeben. Key-Praefix: `cache:` wird automatisch vorangestellt.

2. `async set<T>(key: string, data: T, ttl: number): Promise<void>` — Schreibt `{ data, timestamp: Date.now(), ttl }` als JSON-String in Preferences. Key-Praefix: `cache:`.

3. `isStale(entry: CacheEntry<any>): boolean` — `Date.now() - entry.timestamp > entry.ttl`

4. `async remove(key: string): Promise<void>` — Loescht einen Cache-Key.

5. `async clearAll(): Promise<void>` — Loescht ALLE Keys die mit `cache:` beginnen. Nutzt `Preferences.keys()` um alle Keys zu finden, filtert nach Praefix `cache:`, loescht jeden einzeln mit `Preferences.remove()`.

TTL-Default-Konstanten exportieren (per D-04):
```typescript
export const CACHE_TTL = {
  DASHBOARD: 5 * 60 * 1000,      // 5 Min
  EVENTS: 10 * 60 * 1000,         // 10 Min
  CHAT_ROOMS: 2 * 60 * 1000,      // 2 Min
  CHAT_MESSAGES: 60 * 60 * 1000,   // 1 Std
  STAMMDATEN: 60 * 60 * 1000,     // 1 Std
  PROFILE: 15 * 60 * 1000,        // 15 Min
  BADGES: 30 * 60 * 1000,         // 30 Min

  SETTINGS: 30 * 60 * 1000,       // 30 Min
  TAGESLOSUNG: 24 * 60 * 60 * 1000, // 24 Std

```

Hook-Signatur (per D-06):
```typescript
function useOfflineQuery<T>(
  cacheKey: string,
  fetcher: () => Promise<T>,
  options?: UseOfflineQueryOptions<T>
): UseOfflineQueryResult<T>
```

Options Interface (per D-06):
```typescript
interface UseOfflineQueryOptions<T> {
  ttl?: number;           // Default: 5 * 60 * 1000 (5 Min)
  enabled?: boolean;      // Default: true
  onSuccess?: (data: T) => void;
  onError?: (error: Error) => void;
  select?: (data: T) => T;
}
```

Result Interface (per D-06):
```typescript
interface UseOfflineQueryResult<T> {
  data: T | null;
  loading: boolean;
  error: string | null;
  isStale: boolean;
  isOffline: boolean;
  refresh: () => Promise<void>;
}
```

Implementierung basiert auf der Research-Vorlage aus DEEP-OFFLINE-ANALYSIS.md Section 10:

1. **Initial Load** (useEffect auf [cacheKey, enabled]):
   - Cache lesen via `offlineCache.get(cacheKey)`
   - Wenn Cache vorhanden: `setData(transformed)`, `setLoading(false)`
   - Wenn Cache stale + online: `revalidate()` im Hintergrund
   - Wenn kein Cache + online: `revalidate()` direkt
   - Wenn kein Cache + offline: `setError('Keine Daten verfuegbar (offline)')`, `setLoading(false)` (per D-08)

2. **Revalidation** (useCallback):
   - `fetcher()` ausfuehren
   - Race-Condition-Schutz: `currentKeyRef` pruefen (per D-11)
   - Bei Erfolg: `setData`, `setIsStale(false)`, `offlineCache.set(cacheKey, freshData, ttl)`
   - Bei Fehler + Cache vorhanden: `setIsStale(true)`, Daten behalten (per D-09)
   - Bei Fehler + kein Cache: `setError(message)`, `setLoading(false)`

3. **Network-Listener** (useEffect):
   - `networkMonitor.subscribe()` — bei online-Wechsel: `setIsOffline(false)`, wenn stale: `revalidate()`
   - Bei offline: `setIsOffline(true)`

4. **refresh()** Funktion (per D-10):
   - Nur revalidieren wenn `networkMonitor.isOnline`
   - Fuer IonRefresher und useLiveRefresh Nutzung

WICHTIG: `select` Transformation auf Cache-Daten UND frische Daten anwenden. Cache speichert RAW-Daten (vor select), select wird nur beim Setzen von `data` angewandt.
