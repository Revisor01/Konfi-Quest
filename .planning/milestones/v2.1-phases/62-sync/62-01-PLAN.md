**offlineCache.ts — invalidateAll() hinzufuegen (per D-01, D-02):**

1. Neue async Methode `invalidateAll()` im offlineCache Objekt. Diese setzt den `timestamp` aller Cache-Eintraege auf 0, sodass `isStale()` bei jedem Eintrag `true` zurueckgibt. Das loest bei der naechsten useOfflineQuery-Nutzung automatisch eine Revalidierung aus.

Implementierung:
```typescript
async function invalidateAll(): Promise<void> {
  const { keys } = await Preferences.keys();
  const cacheKeys = keys.filter(k => k.startsWith(CACHE_PREFIX));
  for (const key of cacheKeys) {
    try {
      const result = await Preferences.get({ key });
      if (!result.value) continue;
      const entry = JSON.parse(result.value) as CacheEntry<unknown>;
      entry.timestamp = 0; // Macht isStale() => true
      await Preferences.set({ key, value: JSON.stringify(entry) });
    } catch {
      // Korrupter Eintrag — ignorieren
    }
  }
}
```

2. `invalidateAll` zum Export-Objekt hinzufuegen.

**websocket.ts — koordinierte Reconnect-Sequenz (per D-01, D-02, D-03):**

2. Im `socket.on('connect', ...)` Handler: Die bestehenden `reconnectCallbacks.forEach(cb => cb())` Aufrufe BEIBEHALTEN (Chat-Nachladen etc.), aber DAVOR die koordinierte Sequenz ausfuehren:

```typescript
socket.on('connect', () => {
  if (_hasConnectedOnce) {
    // Koordinierte Sync-Sequenz: flush -> invalidate -> badges -> callbacks
    (async () => {
      try {
        // 1. Queue flushen (Server-State aktuell machen)
        await writeQueue.flush();
        // 2. Cache invalidieren (stale markieren fuer Revalidierung)
        await offlineCache.invalidateAll();
        // 3. Badge-Refresh via CustomEvent (BadgeContext hoert darauf)
        window.dispatchEvent(new CustomEvent('sync:reconnect'));
      } catch (e) {
        console.error('Reconnect sync error:', e);
      }
      // 4. Bestehende Reconnect-Callbacks (Chat-Nachladen etc.)
      reconnectCallbacks.forEach(cb => {
        try { cb(); } catch (e) { console.error('Reconnect callback error:', e); }
      });
    })();
  }
  _hasConnectedOnce = true;
});
```

WICHTIG: Die Sequenz ist async, damit flush() und invalidateAll() awaited werden BEVOR die Callbacks laufen. So ist der Cache stale bevor Chat-Komponenten ihre Reconnect-Handler ausfuehren.
