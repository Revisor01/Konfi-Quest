Die Admin-EventDetailView ist eine grosse Datei (~700+ Zeilen) mit ~82 Inline-Styles. Folgende Umstellungen:

**A) SectionHeader einbauen:**
1. Import hinzufuegen: `import { SectionHeader, ListSection } from '../../shared';`
2. Den bestehenden grossen Header-Block (der aktuell Inline-Styles fuer Hintergrundfarbe, Gradient, Titel, etc. hat) durch einen SectionHeader ersetzen.

Status-Farb-Mapping fuer SectionHeader `colors`-Prop:
```tsx
const getStatusColors = (status: string) => {
  switch (status) {
    case 'upcoming': return { primary: '#007aff', secondary: '#0066d6' };
    case 'open': return { primary: '#34c759', secondary: '#2db84d' };
    case 'closed': return { primary: '#8e8e93', secondary: '#636366' };
    case 'cancelled': return { primary: '#dc3545', secondary: '#c82333' };
    default: return { primary: '#8e8e93', secondary: '#636366' };
  }
};
```

SectionHeader-Aufruf:
```tsx
<SectionHeader
  title={event.name}
  subtitle={getRegistrationStatusText(event)}
  icon={calendar}
  colors={getStatusColors(event.registration_status)}
  stats={[
    { value: event.registered_count, label: 'Gebucht' },
    { value: event.available_spots >= 0 ? event.available_spots : 0, label: 'Frei' },
    { value: event.points, label: 'Punkte' }
  ]}
/>
```

**B) Inline-Styles durch CSS-Klassen ersetzen:**
- Status-Chips (die kleinen farbigen Status-Labels) -> `app-status-chip` + `app-status-chip--{variant}`
- Info-Hinweise am Ende -> `app-info-box` + `app-info-box--neutral`
- Listenelemente die SectionHeader/ListSection nutzen koennten -> ListSection mit passenden Props
- `style={{ margin: '16px' }}` bei IonList -> bleibt (Ionic-Pattern)
- Farbige `app-icon-circle` werden bereits per CSS-Klasse gestylt -> beibehalten
- `app-list-item` Klassen werden bereits genutzt -> beibehalten

**C) Inline-Styles die BLEIBEN duerfen:**
- Dynamische `width` basierend auf Prozentwerten (z.B. Progress-Bars)
- Dynamische `backgroundColor` basierend auf berechneten Werten
- Dynamische `opacity` basierend auf Status
- `--background: transparent` auf IonItems (Ionic CSS Custom Properties)

**D) Listen-Sektionen auf ListSection umstellen:**
Die Teilnehmer-Liste, Warteliste und Event-Details-Sektion sollten ListSection nutzen wo sinnvoll. Falls die Struktur zu komplex fuer ListSection ist (z.B. Attendance-Buttons), kann die bestehende IonList-Struktur bleiben -- aber mit moeglichst wenig Inline-Styles.
