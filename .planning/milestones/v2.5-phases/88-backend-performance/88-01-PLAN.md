Den SQL-Query in GET /rooms (Zeilen 383-420) erweitern: einen LATERAL Join hinzufuegen, der fuer Direct-Rooms den Namen des anderen Teilnehmers direkt in der Haupt-Query ermittelt:

```sql
LEFT JOIN LATERAL (
  SELECT u.display_name as direct_name
  FROM chat_participants dp
  JOIN users u ON dp.user_id = u.id
  WHERE dp.room_id = r.id
    AND NOT (dp.user_id = $1 AND dp.user_type = $2)
  LIMIT 1
) dm ON r.type = 'direct'
```

Diesen Join in den SELECT aufnehmen:
```sql
COALESCE(CASE WHEN r.type = 'direct' THEN dm.direct_name ELSE r.name END, r.name) as name
```

Hinweis: r.* nicht mehr verwenden (da name ueberschrieben), sondern explizit die benoetigen Felder aus r.* aufzaehlen ODER r.* beibehalten und name als eigenes Feld danach selektieren (spaetere Felder ueberschreiben frueheres r.name in pg-Ergebnis nicht automatisch -- deshalb COALESCE-Ausdruck als `name`-Alias und sicherstellen dass kein doppeltes `name` im SELECT ist).

Den Promise.all-Block (Zeilen 426-441) komplett entfernen. `finalRooms` durch direkte Verwendung von `rooms` ersetzen:


GROUP BY entsprechend erweitern falls dm.direct_name im SELECT referenziert wird (oder Subquery-Ergebnis ist skalarer Wert aus LATERAL, kein GROUP BY noetig).

<task type="auto">
  <name>Task 2: Korrelierte Subquery durch LATERAL Join ersetzen (PERF-02)</name>
  <files>backend/routes/chat.js</files>
  <action>
Die ORDER BY-Klausel in GET /rooms (Zeile 419) enthaelt eine korrelierte Subquery:
```sql
ORDER BY (SELECT created_at FROM chat_messages WHERE room_id = r.id AND deleted_at IS NULL ORDER BY created_at DESC LIMIT 1) DESC NULLS LAST
```

Diese durch das bereits im Query vorhandene `last_message` LATERAL-Subselect-Ergebnis ersetzen. Da `last_message` als json_build_object vorliegt, einen dedizierten LATERAL Join fuer den Sortier-Timestamp einfuegen:

```sql
LEFT JOIN LATERAL (
  SELECT created_at as last_message_at
  FROM chat_messages
  WHERE room_id = r.id AND deleted_at IS NULL
  ORDER BY created_at DESC
  LIMIT 1
) lm ON true
```

ORDER BY dann:
```sql
ORDER BY lm.last_message_at DESC NULLS LAST
```

Damit entfaellt die korrelierte Subquery komplett. Der neue LATERAL Join wird einmalig ausgefuehrt und das Ergebnis fuer SELECT und ORDER BY wiederverwendet.
