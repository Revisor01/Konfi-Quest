**1. QRScannerModal.tsx (Konfi - Fullscreen Kamera-Scanner):**

Props: `{ onClose: () => void; onSuccess: (eventId: number, eventName: string) => void }`

Aufbau:
- IonPage mit IonHeader (Titel "QR-Code scannen", Close-Button) und IonContent
- Video-Element (ref) fuer Kamera-Preview, fullscreen innerhalb Content
- Scanner-Overlay mit halbtransparentem Rahmen (CSS: scan-region Indikator)
- Inline-Banner fuer Fehler (rot) und Info (blau) am oberen Rand des Kamera-Feeds
- Permission-Denied Fallback-Ansicht mit Erklaerung und Link zu Einstellungen

Logik:
- Worker-Path Setup: `import QrScannerWorkerPath from 'qr-scanner/qr-scanner-worker.min.js?url'; QrScanner.WORKER_PATH = QrScannerWorkerPath;`
- useEffect: Scanner erstellen, starten, bei Cleanup: `scanner.destroy()`
- `scanning` State-Flag: verhindert doppelte Verarbeitung
- Bei Scan-Ergebnis:
  - `setScanning(true)` sofort
  - `scanner.stop()` sofort (verhindert weitere Scans waehrend Request)
  - API-Call: `api.post('/events/qr-checkin', { token: result.data })`
  - Bei Erfolg (200, !already_checked_in): `onSuccess(eventId, eventName)` aufrufen
  - Bei already_checked_in (200): Blauen Info-Banner zeigen ("Du bist bereits eingecheckt"), nach 2s Scanner wieder starten, `setScanning(false)`
  - Bei Fehler (4xx): Roten Error-Banner zeigen mit error-Text aus Response, nach 3s Scanner wieder starten, `setScanning(false)`
  - Bei Netzwerk-Fehler: "QR-Code konnte nicht verarbeitet werden"

CSS (inline oder im Modal):
- Video: 100% width/height, object-fit: cover
- Banner: position absolute, top: 60px (unter Header), left/right: 16px, padding: 12px, border-radius: 8px, z-index: 10
- Fehler-Banner: background var(--ion-color-danger), color white
- Info-Banner: background var(--ion-color-primary), color white
- Permission-Denied: zentrierter Text mit IonIcon (cameraOutline)

**2. QRDisplayModal.tsx (Admin - Fullscreen QR-Anzeige mit Live-Zaehler):**

Props: `{ eventId: number; eventName: string; eventDate: string; onClose: () => void }`

Aufbau:
- IonPage mit IonHeader (Titel "QR-Code", Close-Button, Drucken-Button als IonButton mit printOutline Icon)
- IonContent mit weissem Hintergrund, zentriertem Layout:
  - Event-Name (gross, fett)
  - Event-Datum/Uhrzeit
  - QR-Code Bild (gross, 300x300 auf Mobile, 400x400 auf Desktop)
  - Live-Zaehler: "X / Y eingecheckt" (gross, unter QR-Code)
  - Hinweistext: "Konfis scannen diesen QR-Code mit der App zum Einchecken"

Logik:
- loadQR: `api.post(`/events/${eventId}/generate-qr`)` -> Token -> `QRCode.toDataURL(token, { width: 512, margin: 2, errorCorrectionLevel: 'H' })` -> State
- Polling (useEffect + setInterval, 10 Sekunden): `api.get(`/events/${eventId}/attendance-count`)` -> `{ checked_in, total }` -> State
- Cleanup: clearInterval bei Modal-Close
- **Drucken:** `window.print()` aufrufen. CSS @media print Regeln:
  - Header und Toolbar ausblenden (display: none)
  - QR-Code zentriert, gross (80vw max)
  - Event-Name und Datum sichtbar
  - Hinweistext sichtbar
  - Zaehler ausblenden (nicht relevant auf Papier)
  - Weisser Hintergrund erzwingen

CSS:
- Container: display flex, flex-direction column, align-items center, justify-content center, min-height 100%, padding 24px
- QR-Image: max-width 300px (mobil) / 400px (>768px), border: 2px solid var(--border-color), border-radius 12px, padding 16px, background white
- Event-Name: font-size 1.4rem, font-weight 700, text-align center, margin-bottom 8px
- Zaehler: font-size 1.8rem, font-weight 700, margin-top 24px
- Hinweistext: color var(--text-secondary), font-size 0.9rem, margin-top 16px

WICHTIG:
