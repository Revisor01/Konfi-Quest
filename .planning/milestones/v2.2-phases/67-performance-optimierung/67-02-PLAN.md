In jeder Route-Datei SELECT * durch die tatsaechlich benoetigten Spalten ersetzen. Vorgehen pro Query:

1. Die Zeile mit SELECT * lesen
2. Schauen welche Felder des Ergebnisses im Code danach verwendet werden (destructuring, property access)
3. SELECT * durch `SELECT col1, col2, col3` ersetzen mit genau den genutzten Spalten

**konfi.js (7 Stellen):**
- `SELECT * FROM levels` (2x) — ersetzen durch `SELECT id, name, title, icon, color, points_required, sort_order FROM levels`
- `SELECT * FROM activity_requests` (3x) — ersetzen durch die genutzten Spalten (id, konfi_id, activity_id, status, client_id, etc.)
- `SELECT * FROM event_bookings` (2x) — ersetzen durch `SELECT id, user_id, event_id, status, booking_date, timeslot_id FROM event_bookings`
- `SELECT * FROM event_timeslots` — ersetzen durch die genutzten Spalten

**events.js (4 Stellen):**
- `SELECT * FROM events WHERE id = $1` (3x, davon 2x mit FOR UPDATE) — ersetzen durch alle genutzten Event-Spalten
- `SELECT * FROM event_timeslots` — ersetzen durch genutzte Timeslot-Spalten
- WICHTIG: Bei `FOR UPDATE` Queries ALLE Spalten der Tabelle auflisten (da das gesamte Row gelockt und aktualisiert wird)

**auth.js (3 Stellen):**
- `SELECT * FROM users WHERE id = $1` — ersetzen durch `SELECT id, display_name, username, role_id, organization_id, profile_image FROM users`
- `SELECT * FROM invite_codes` — ersetzen durch genutzte Spalten
- `SELECT * FROM password_resets` — ersetzen durch genutzte Spalten

**badges.js (2 Stellen):**
- `SELECT * FROM custom_badges WHERE is_active = true` (2x) — ersetzen durch die genutzten Badge-Spalten

KONSERVATIV: Wenn unklar welche Spalten genutzt werden (z.B. bei `res.json(row)` wo der gesamte Row zurueckgegeben wird), dann ALLE Spalten der Tabelle explizit auflisten statt SELECT * zu behalten. So ist dokumentiert welche Spalten existieren.
