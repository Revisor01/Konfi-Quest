Drei Stellen in activities.js muessen in Transaktionen gewrapped werden:

1. **POST /assign-activity** (Zeile ~398-428): INSERT konfi_activities + UPDATE konfi_profiles in BEGIN/COMMIT wrappen. Bei Fehler ROLLBACK. Push-Notification und liveUpdate NACH dem COMMIT senden.

2. **POST /assign-bonus** (Zeile ~433-466): INSERT bonus_points + UPDATE konfi_profiles in BEGIN/COMMIT wrappen. Bei Fehler ROLLBACK. Push-Notification und liveUpdate NACH dem COMMIT senden.

3. **PUT /requests/:id** (Zeile ~284-389): Die approved-Branch hat INSERT konfi_activities + UPDATE konfi_profiles ohne Transaktion. Den gesamten try-Block in BEGIN/COMMIT wrappen. Photo-Loeschung und Notifications NACH dem COMMIT senden (nicht-kritisch).

4. **PUT /requests/:id/reset** (Zeile ~230-280): UPDATE konfi_profiles (Punkte abziehen) + DELETE konfi_activities + UPDATE activity_requests in BEGIN/COMMIT wrappen. ZUSAETZLICH: Bei Punkte-Abzug GREATEST(0, ...) verwenden:
   ```sql
   UPDATE konfi_profiles SET {pointField} = GREATEST(0, {pointField} - $1) WHERE user_id = $2
   ```

Fuer alle Transaktionen: `const client = await db.connect()` verwenden, damit BEGIN/COMMIT/ROLLBACK auf derselben Connection laufen. Am Ende IMMER `client.release()` in einem finally-Block.

Pattern:
```javascript
const client = await db.connect();
try {
  await client.query('BEGIN');
  // ... alle Queries mit client.query statt db.query
  await client.query('COMMIT');
} catch (err) {
  await client.query('ROLLBACK');
  throw err;
} finally {
  client.release();
}
```

WICHTIG: checkAndAwardBadges benoetigt `db` (den Pool), nicht `client`. Badge-Check NACH dem COMMIT ausfuehren, da er eigene Queries macht.
