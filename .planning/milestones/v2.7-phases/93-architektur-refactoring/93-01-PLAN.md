**ARCH-06 — chatUtils dynamischer Admin:**

In `backend/utils/chatUtils.js`, Zeile 29 (`1 // Default admin user ID`) ersetzen:

Vor dem Jahrgang-Loop eine Hilfsfunktion `getFirstActiveAdmin(db, organizationId)` definieren (oder inline-Query), die den ersten aktiven Admin der Organisation holt:

```sql
SELECT u.id FROM users u
JOIN roles r ON u.role_id = r.id
WHERE r.name = 'admin' AND u.organization_id = $1 AND u.is_active = true
ORDER BY u.id ASC LIMIT 1
```

Falls kein Admin existiert (leere Organisation), Fallback auf ID 1 mit console.warn. Den Query-Result als `created_by` Parameter verwenden statt der hardcodierten `1`.

Die Funktion soll pro Organisation einmal abgefragt werden (nicht pro Jahrgang), also den Admin-Lookup VOR den Jahrgang-Loop heben und in einer Map `orgId -> adminId` cachen.

**ARCH-07 — bookingUtils.js erstellen:**

Neue Datei `backend/utils/bookingUtils.js` erstellen mit folgenden exportierten Funktionen:

1. `async function checkExistingBooking(client, userId, eventId)` — Prueft ob bereits gebucht, gibt booking oder null zurueck

2. `async function getEventWithCounts(client, eventId, orgId, options)` — Laedt Event mit confirmed_count und waitlist_count (FOR UPDATE). `options.excludeTeamers` steuert ob Teamer-Bookings gegen Kapazitaet zaehlen

3. `async function determineBookingStatus(event, confirmedCount, waitlistCount, maxCapacity)` — Gibt 'confirmed', 'waitlist', oder {error, status} zurueck

4. `async function promoteFromWaitlist(db, eventId, timeslotId)` — Rueckt ersten Wartelisten-Eintrag nach (timeslot-aware), gibt promotedUserId oder null zurueck

5. `async function validateRegistrationWindow(event)` — Prueft ob Anmeldezeitraum offen ist, gibt {valid, error} zurueck

Pattern: `module.exports = { checkExistingBooking, getEventWithCounts, determineBookingStatus, promoteFromWaitlist, validateRegistrationWindow };`

**ARCH-08 — fetcherRef Pattern:**

In `frontend/src/hooks/useOfflineQuery.ts`:

1. Neuen Ref fuer den Fetcher anlegen (gleich nach den bestehenden Refs auf Zeile ~54):
```typescript
const fetcherRef = useRef(fetcher);
fetcherRef.current = fetcher;
```

2. In `revalidate` useCallback (Zeile 56-86):
   - `fetcher()` auf Zeile 58 ersetzen durch `fetcherRef.current()`
   - `fetcher` aus dem Dependency-Array entfernen: `[cacheKey, ttl]` statt `[cacheKey, fetcher, ttl]`

Das ist dasselbe Pattern wie bei `onSuccessRef`, `onErrorRef`, `selectRef` — die bereits auf Zeilen 49-54 genauso implementiert sind. Der Fetcher wird via Ref getrackt, sodass der useCallback stabil bleibt und nur bei cacheKey- oder ttl-Aenderung neu erstellt wird.

KEIN weiterer Umbau noetig — die restliche Hook-Logik (useEffect Dependencies, refresh, Network-Listener) funktioniert bereits korrekt, weil sie auf `revalidate` referenzieren, das jetzt stabil ist.
