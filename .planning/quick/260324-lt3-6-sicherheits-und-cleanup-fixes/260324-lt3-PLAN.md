    **Fix 3 — validatePassword in users.js einbinden:**

    Am Anfang von users.js (nach den bestehenden requires) importieren:
    ```javascript
    const { validatePassword } = require('../utils/passwordUtils');
    ```

    In `validateResetPassword` (Zeile 35-39): Die bestehende Validierung:
    ```javascript
    body('password').isLength({ min: 6 }).withMessage('Passwort muss mindestens 6 Zeichen lang sein'),
    ```
    ersetzen durch:
    ```javascript
    body('password').isLength({ min: 8 }).withMessage('Passwort muss mindestens 8 Zeichen lang sein'),
    ```

    Im Handler `PUT /:id/reset-password` (nach `const { password } = req.body;`, Zeile ~547), den bestehenden Check:
    ```javascript
    if (!password || password.length < 6) {
      return res.status(400).json({ error: 'Passwort muss mindestens 6 Zeichen lang sein' });
    }
    ```
    ersetzen durch:
    ```javascript
    const passwordError = validatePassword(password);
    if (passwordError) {
      return res.status(400).json({ error: passwordError });
    }
    ```

    Beim Create-Handler `POST /` (vor `const passwordHash = await bcrypt.hash(...)`, Zeile ~169) einfügen:
    ```javascript
    const passwordError = validatePassword(password);
    if (passwordError) {
      return res.status(400).json({ error: passwordError });
    }
    ```

    **Fix 4 — invalidateUserCache nach Rollenänderung:**

    In users.js den Import von rbac.js hinzufügen (nach dem validatePassword-Import):
    ```javascript
    const { invalidateUserCache } = require('../middleware/rbac');
    ```

    Im PUT-Handler (Update User, nach dem Block der Socket.io-Trennung bei Rollenänderung, ca. Zeile 260-270):
    Nach der Stelle wo `role_id !== undefined && io` geprüft wird, direkt danach einfügen:
    ```javascript
    // Cache invalidieren damit neues Token sofort wirkt
    if (role_id !== undefined) {
      invalidateUserCache(parseInt(id));
    }
    ```

    **Fix 5 — setImmediate Bug in server.js:**

    Zeile 527 in server.js:
    ```javascript
    setImmediate(initializeChatRooms(db));
    ```
    ersetzen durch:
    ```javascript
    setImmediate(() => initializeChatRooms(db));
    ```
    Begründung: `initializeChatRooms(db)` wird sofort aufgerufen und gibt ein Promise zurück — das Promise wird als Callback übergeben, nicht die Funktion selbst. Arrow-Function-Wrapper stellt korrekte Lazy-Ausführung sicher.

    **Fix 6 — Legacy data/-Verzeichnis in server.js entfernen:**

    ```javascript
    const dataDir = path.join(__dirname, 'data');
    if (!fs.existsSync(dataDir)) {
      fs.mkdirSync(dataDir, { recursive: true });
    }
    ```
    Auch den Kommentar-Block `// DATA DIRECTORIES SETUP` darüber (Zeilen 420-422) entfernen.
    Danach prüfen ob `fs` noch an anderer Stelle in server.js verwendet wird — falls nicht, den `fs`-Import (Zeile 8) entfernen.
